  <action>
1. Install ESLint v9 + eslint-plugin-boundaries + TypeScript parser:
   ```bash
   npm install --save-dev eslint eslint-plugin-boundaries @typescript-eslint/parser
   ```

2. Create `eslint.config.js` (flat config, ESLint v9) at project root:
   ```javascript
   import boundaries from 'eslint-plugin-boundaries'
   import tsParser from '@typescript-eslint/parser'

   export default [
     {
       files: ['src/**/*.{ts,tsx}'],
       languageOptions: {
         parser: tsParser,
         parserOptions: {
           ecmaFeatures: { jsx: true },
         },
       },
       plugins: { boundaries },
       settings: {
         'boundaries/elements': [
           { type: 'module', pattern: ['src/modules/*'] },
           { type: 'shared-component', pattern: ['src/components/*'] },
           { type: 'page', pattern: ['src/pages/*'] },
           { type: 'lib', pattern: ['src/lib/*'] },
         ],
       },
       rules: {
         'boundaries/element-types': [
           'error',
           {
             default: 'allow',
             rules: [
               {
                 from: ['module'],
                 disallow: ['module'],
                 // Modules cannot import from other modules' internals
                 // Registry (src/modules/registry.ts) is at the modules root, not inside a module subfolder
               },
             ],
           },
         ],
       },
     },
   ]
   ```

   NOTE: Use `default: 'allow'` approach — safer for existing codebase. The rule only blocks cross-module imports (module A importing from module B). Importing from shared components, lib, and pages is still allowed (needed for wrapper manifests that import page components).

3. Update `package.json` lint script:
   Change `"lint": "tsc --noEmit"` to `"lint": "tsc --noEmit && eslint src/"`.
   For ESLint v9 flat config, do NOT use `--ext` flag (deprecated).

4. Run `npm run lint` and verify it passes. If eslint-plugin-boundaries flags any existing imports as violations, investigate:
   - The wrapper manifests import from `src/pages/*` — this is `module` importing from `page`, which should be allowed by `default: 'allow'`
   - If false positives appear, adjust the `boundaries/elements` patterns or add an `allow` exception

IMPORTANT per STATE.md blocker: "eslint-plugin-boundaries exact config syntax needs verification during Phase 29 execution" — if the plugin's flat config syntax differs from what's documented in RESEARCH.md, adapt accordingly. The key behaviors to preserve: (1) cross-module imports are errors, (2) everything else is allowed.
