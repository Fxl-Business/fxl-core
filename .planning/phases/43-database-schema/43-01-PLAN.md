  <action>
Create `supabase/migrations/007_documents.sql` following the exact pattern of 005 and 006 migrations.

**Table definition** — `public.documents`:
- `id` uuid PRIMARY KEY DEFAULT gen_random_uuid()
- `title` text NOT NULL
- `badge` text NOT NULL — no CHECK constraint (per user decision: keep flexible for new categories)
- `description` text NOT NULL DEFAULT ''
- `slug` text NOT NULL UNIQUE — each document has a unique slug, this is how the app resolves routes
- `parent_path` text NOT NULL — maps to directory structure (e.g., 'docs/processo/fases/')
- `body` text NOT NULL — full markdown content including custom tags
- `sort_order` integer NOT NULL DEFAULT 0 — ordering within parent_path
- `created_at` timestamptz NOT NULL DEFAULT now()
- `updated_at` timestamptz NOT NULL DEFAULT now()

**RLS** — Enable row level security, create 4 anon-permissive policies:
- `anon_read_documents` — FOR SELECT TO anon USING (true)
- `anon_insert_documents` — FOR INSERT TO anon WITH CHECK (true)
- `anon_update_documents` — FOR UPDATE TO anon USING (true)
- `anon_delete_documents` — FOR DELETE TO anon USING (true)

**Indexes:**
- `idx_documents_parent_path` — B-tree on parent_path (sidebar/navigation queries)
- `idx_documents_parent_path_order` — Composite B-tree on (parent_path, sort_order) (ordered listing within sections)

Note: slug index is implicitly created by the UNIQUE constraint. No need for explicit idx_documents_slug.

**SQL comment header** — Match style of existing migrations:
```sql
-- Migration: Documents table for dynamic doc content (from docs/ filesystem)
-- Stores title, badge, description (frontmatter) + body (markdown) + navigation structure.
-- All access through anon role (Clerk handles auth externally).
```

Do NOT add: search_vec/tsvector, is_published, metadata JSONB, or any columns not listed above. Schema must be minimal, mirroring exactly what exists in frontmatter + structure today.
