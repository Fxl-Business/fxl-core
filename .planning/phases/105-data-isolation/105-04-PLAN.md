# Plan 105-04: TypeScript Verification + Fix Callers

## Goal

Run `npx tsc --noEmit` and fix any type errors introduced by the changes in Plans 105-01 through 105-03. Primary concern: `createTask` now requires `org_id` â€” all callers must be updated. Secondary concern: `invalidateDocsCache` import in `useOrgTokenExchange` must resolve cleanly.

## Context

Plans 105-01 through 105-03 make breaking changes to `CreateTaskParams` (new required field `org_id`). Any component that calls `createTask` without passing `org_id` will fail TypeScript. This plan is the verification + fix sweep.

### Task 2: Fix createTask callers

<read_first>
- src/modules/tasks/services/tasks-service.ts
- Any file with createTask calls (found via grep in tsc output or explicit search)
</read_first>

<action>
Find all callers of `createTask`:
```bash
grep -r "createTask" src/ --include="*.tsx" --include="*.ts" -n
```

For each caller:
1. Read the file
2. Check if it already imports `useActiveOrg` from `@platform/tenants/useActiveOrg`
3. If not, add the import
4. Destructure `activeOrg` from `useActiveOrg()` in the component
5. Add `org_id: activeOrg?.id ?? ''` to the `createTask(...)` call

Pattern:
```typescript
// Before:
const task = await createTask({ title, description, status, priority })

// After:
const { activeOrg } = useActiveOrg()
// ...
const task = await createTask({ title, description, status, priority, org_id: activeOrg?.id ?? '' })
```

Note: `useActiveOrg()` must be called at the top of the component (hook rules), not inside event handlers. The destructured `activeOrg` is then used in the event handler callback.
</action>

<acceptance_criteria>
- All `createTask` call sites include `org_id` field
- Files with updated callers import `useActiveOrg` from `@platform/tenants/useActiveOrg`
- `useActiveOrg()` is called at component top level (not inside callbacks)
- grep `createTask({` in src/ shows no missing org_id
</acceptance_criteria>
