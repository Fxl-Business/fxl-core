Create the module folder structure: `src/modules/knowledge-base/types/`, `src/modules/knowledge-base/hooks/`, `src/modules/knowledge-base/components/`, `src/modules/knowledge-base/pages/`.

**src/modules/knowledge-base/types/kb.ts:**
- Re-export `KBEntry` and `KBFilters` from `@/lib/kb-service`
- Define `KBEntryType = 'bug' | 'decision' | 'pattern' | 'lesson'` as a union type (discriminated, not magic strings — per STATE.md decision)
- Export `KB_ENTRY_TYPES` constant array of all 4 types with labels in Portuguese: `[{ value: 'bug', label: 'Bug' }, { value: 'decision', label: 'Decisao' }, { value: 'pattern', label: 'Padrao' }, { value: 'lesson', label: 'Licao' }]`
- Export `ADR_TEMPLATE` constant string with `## Context`, `## Decision`, `## Consequences` sections (Portuguese text, per KB-06 and RESEARCH pattern 4)

**src/modules/knowledge-base/hooks/useKBEntries.ts:**
- Import `listKBEntries` and `KBFilters` from `@/lib/kb-service`
- Accept `filters: KBFilters` parameter
- Return `{ entries, loading, error, refetch }` — standard hook shape
- Use `useEffect` with cancellation pattern (see RESEARCH pattern 1)
- CRITICAL: Serialize tags array with `JSON.stringify(filters.tags)` in useEffect dependency array to prevent infinite re-renders (RESEARCH pitfall 2)
- `refetch` is a function that re-triggers the fetch (increment a counter state)

**src/modules/knowledge-base/hooks/useKBEntry.ts:**
- Import `getKBEntry` from `@/lib/kb-service`
- Accept `id: string | undefined` parameter
- Return `{ entry, loading, error }`
- Skip fetch if id is undefined (for create mode in form page)
- Use same cancellation pattern

**src/modules/knowledge-base/hooks/useKBSearch.ts:**
- Import `searchKBEntries` from `@/lib/kb-service`
- Accept `query: string` parameter
- Return `{ results, loading, error }`
- Only trigger fetch when query is at least 2 characters (debounce not needed — user submits explicitly)
- If query is empty or less than 2 chars, return empty results without fetching

IMPORTANT: All hooks must declare hooks before any conditional returns (memory rule: hooks-before-early-returns).

If `kb-service.ts` does not yet exist from Phase 30, create a minimal stub at `src/lib/kb-service.ts` with the typed function signatures from the interfaces block above, each throwing `new Error('Not implemented — Phase 30 pending')`.
