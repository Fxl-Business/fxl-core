**src/modules/knowledge-base/pages/KBFormPage.tsx (KB-04, KB-06):**
- Dual mode: create (/knowledge-base/new) and edit (/knowledge-base/:id/edit)
- Detect mode via `useParams()`: if `id` exists, it's edit mode — fetch entry with `useKBEntry(id)`
- Form state: `{ title: string, body: string, type: KBEntryType, tags: string, clientSlug: string }` — tags stored as comma-separated string in the form input, parsed to array on submit
- Form fields (use shadcn components):
  1. **Titulo** — Input, required, placeholder "Titulo da entrada"
  2. **Tipo** — Select with 4 options from KB_ENTRY_TYPES constant (Bug, Decisao, Padrao, Licao)
  3. **Corpo** — Textarea with min-h-[300px] for markdown body, placeholder "Conteudo em Markdown..."
  4. **Tags** — Input, placeholder "Tags separadas por virgula (ex: supabase, auth, deploy)"
  5. **Cliente** — Input, placeholder "Slug do cliente (ex: financeiro-conta-azul)", optional
- ADR template injection (KB-06): When type changes to 'decision' AND body is empty (or only whitespace), set body to ADR_TEMPLATE from types/kb.ts. Guard: `if (formData.type === 'decision' && !formData.body.trim())` — do NOT overwrite existing content on edit (RESEARCH pitfall 5).
- On submit:
  - Parse tags: `formData.tags.split(',').map(t => t.trim()).filter(Boolean)` (RESEARCH pitfall 4)
  - In create mode: call `createKBEntry(...)`, on success show `toast.success('Entrada criada')` and navigate to `/knowledge-base`
  - In edit mode: call `updateKBEntry(id, ...)`, on success show `toast.success('Entrada atualizada')` and navigate to `/knowledge-base/${id}`
  - On error: `toast.error('Erro ao salvar: ' + error.message)`
- Show "Cancelar" link back to /knowledge-base (or /knowledge-base/:id on edit)
- Page title: "Nova entrada" or "Editar entrada" based on mode
- Page wrapper: `max-w-3xl mx-auto px-6 py-8`
