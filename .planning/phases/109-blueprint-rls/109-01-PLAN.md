### Task 1: Verify RLS Policies via SQL Audit

<read_first>
- supabase/migrations/013_remove_anon_fallback.sql (source of truth for expected RLS policy text)
- supabase/migrations/017_data_recovery.sql (confirms org_id data is correctly set)
</read_first>

<action>
Run the following SQL via the Supabase MCP tool (mcp__supabase__execute_sql) to audit the current state:

**Query 1 — Confirm RLS is enabled:**
```sql
SELECT relname AS table_name, relrowsecurity AS rls_enabled, relforcerowsecurity AS rls_forced
FROM pg_class
WHERE relname IN ('blueprint_configs', 'briefing_configs')
AND relnamespace = 'public'::regnamespace;
```
Expected: `rls_enabled = true` for both rows.

**Query 2 — Confirm policy exists and has correct expression:**
```sql
SELECT tablename, policyname, cmd, roles, qual
FROM pg_policies
WHERE tablename = 'blueprint_configs';
```
Expected: one row with `policyname = 'blueprint_configs_org_access'`, `cmd = 'ALL'`, `roles = '{anon,authenticated}'`, and `qual` containing both `super_admin` and `org_id` checks.

**Query 3 — Confirm no blueprints table exists separately:**
```sql
SELECT relname FROM pg_class
WHERE relname = 'blueprints'
AND relnamespace = 'public'::regnamespace;
```
Expected: zero rows (no separate blueprints table).

**Query 4 — Confirm blueprint_configs rows have real org_id (not placeholder):**
```sql
SELECT COUNT(*) as placeholder_count
FROM public.blueprint_configs
WHERE org_id = 'org_fxl_default';
```
Expected: 0 (all rows were updated by migration 017).
</action>

- Query 2 returns row with `policyname = 'blueprint_configs_org_access'` and `cmd = 'ALL'`
- Query 2 `qual` field contains both `'super_admin'` and `'org_id'` substrings
- Query 3 returns zero rows (no separate blueprints table)
- Query 4 returns `placeholder_count = 0`
