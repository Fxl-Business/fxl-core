<task type="auto">
  <name>Task 1: Build KanbanBoard page with 4 status columns</name>
  <files>src/modules/tasks/pages/KanbanBoard.tsx</files>
  <action>
Create the kanban board page that groups tasks into 4 columns by status.

**Layout:**
- Page header: "Kanban" title with a "Lista" secondary Button linking to `/tarefas` and a "Nova Tarefa" primary Button linking to `/tarefas/new`
- Grid: `grid grid-cols-1 sm:grid-cols-2 xl:grid-cols-4 gap-4` (responsive — see research pitfall about overflow on narrow screens)
- Each column: header with status label (10px uppercase tracking-widest font-bold text-slate-500) + task count, then a container with rounded-xl border bg-slate-50 dark:bg-slate-900 padding, then TaskCard components stacked vertically
- Empty column: "Vazio" centered in muted text

**Data fetching:**
Use `useTasks()` hook with no filters (fetch all tasks). Group tasks by status using `STATUS_ORDER.map(status => tasks.filter(t => t.status === status))`.

**Status change handler (optimistic update):**
When a TaskCard's status badge is clicked, receive `(taskId, newStatus)`:
1. Immediately update local tasks state: `setTasks(prev => prev.map(t => t.id === taskId ? {...t, status: newStatus} : t))`
2. Call `updateTaskStatus(taskId, newStatus)` from tasks-service
3. On error: `toast.error('Erro ao atualizar status')` and call `refetch()` to rollback

Note: `useTasks` returns tasks array — to do optimistic updates, manage a local copy via useState initialized from useTasks and synced via useEffect. Or: destructure tasks from useTasks, copy into local state, and use local state for rendering. ALL hooks at top (critical rule).

**TaskCard integration:**
Reuse the TaskCard component from Plan 01. Pass `onStatusChange` prop. In the kanban view, also render the DocumentarButton for tasks with status 'done' (add it as a slot in TaskCard or render it below each done-status TaskCard).

If Plan 01 registered the kanban route with a lazy import placeholder, replace that placeholder with the actual KanbanBoard import in App.tsx. If App.tsx already imports it correctly, no change needed.
  </action>

  </verify>
