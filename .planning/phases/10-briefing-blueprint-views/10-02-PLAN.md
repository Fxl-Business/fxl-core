  <action>
    1. Create `tools/wireframe-builder/lib/blueprint-text.ts` -- pure function module that extracts display data from a BlueprintConfig.

    Define types:
    ```typescript
    type SectionSummary = {
      type: BlueprintSection['type']
      label: string          // from getSectionLabel()
      title?: string         // section.title if present
      keyFields: string[]    // extracted per type
    }
    type ScreenSummary = {
      id: string
      label: string
      filters?: string[]     // filter labels if present
      periodType?: string
      sections: SectionSummary[]
    }
    type BlueprintSummary = {
      clientName: string
      description?: string
      screens: ScreenSummary[]
    }
    ```

    Export `extractBlueprintSummary(config: BlueprintConfig): BlueprintSummary`

    Key field extraction function `extractKeyFields(section: BlueprintSection): string[]` using switch on section.type. MUST handle all 21 section types. Use SECTION_REGISTRY via getSectionLabel() for human-readable type labels. Fallback for any type: return [] (never crash on unknown fields).

    Per-type key field extraction:
    - kpi-grid: section.items.map(i => i.label)
    - bar-line-chart: [`Tipo: ${section.chartType}`, section.title].filter(Boolean)
    - donut-chart: [section.title].filter(Boolean)
    - waterfall-chart: [section.title].filter(Boolean)
    - pareto-chart: [section.title].filter(Boolean)
    - calculo-card: [section.label, section.value].filter(Boolean)
    - data-table: section.columns.map(c => c.label)
    - drill-down-table: [section.title, ...section.levels.map(l => l.label)].filter(Boolean)
    - clickable-table: [section.title, ...section.columns.map(c => c.label)].filter(Boolean)
    - saldo-banco: [section.title].filter(Boolean)
    - manual-input: [section.title].filter(Boolean)
    - upload-section: [section.title].filter(Boolean)
    - config-table: [section.title, ...section.columns.map(c => c.label)].filter(Boolean)
    - info-block: [section.title].filter(Boolean)
    - chart-grid: [section.title].filter(Boolean)
    - settings-page: [section.title, ...section.groups.map(g => g.label)].filter(Boolean)
    - form-section: [section.title, ...section.fields.map(f => f.label)].filter(Boolean)
    - filter-config: [section.title, ...section.filters.map(f => f.label)].filter(Boolean)
    - stat-card: [section.title, section.value].filter(Boolean)
    - progress-bar: [section.title, `${section.value}%`].filter(Boolean)
    - divider: []
    - default: []

    Import BlueprintSection type from '../types/blueprint' and getSectionLabel from './section-registry'.

    2. Create `tools/wireframe-builder/lib/blueprint-export.ts` -- pure function that converts BlueprintConfig to a markdown string.

    Export `exportBlueprintMarkdown(config: BlueprintConfig): string`

    Uses extractBlueprintSummary internally. Generates markdown in this format:
    ```
    # Blueprint: {clientName}

    {description if present}

    ## Telas

    ### {screen.label}
    {Filtros: filter1, filter2 -- if filters present}
    {Periodo: periodType -- if present}

    | Secao | Tipo | Campos Principais |
    |-------|------|-------------------|
    | {title or '-'} | {label} | {keyFields.join(', ') or '-'} |
    ...

    ---

    ### {next screen}
    ...
    ```

    Also export `downloadMarkdown(content: string, filename: string): void` -- browser file download using Blob + URL.createObjectURL pattern (5 lines, no dependency needed).
  </action>

  </verify>
