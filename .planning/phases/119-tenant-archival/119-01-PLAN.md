**Part 3 — Recreate RLS policies with `archived_at IS NULL` on non-super-admin branch.**

The pattern for each of the 7 FOR ALL tables (tenant_modules, comments, share_tokens, blueprint_configs, briefing_configs, knowledge_entries, tasks) is:

```sql
CREATE POLICY "{table}_org_access" ON public.{table}
  FOR ALL TO anon, authenticated
  USING (
    COALESCE(
      nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'super_admin',
      'false'
    ) = 'true'
    OR
    (
      (nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'org_id') = org_id
      AND archived_at IS NULL
    )
  )
  WITH CHECK (
    COALESCE(
      nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'super_admin',
      'false'
    ) = 'true'
    OR
    (
      (nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'org_id') = org_id
      AND archived_at IS NULL
    )
  );
```

Apply this pattern to all 7 tables: tenant_modules, comments, share_tokens, blueprint_configs, briefing_configs, knowledge_entries, tasks.

For clients and projects tables — check if they already have RLS policies. If yes, drop and recreate with same archived_at pattern. If no RLS policies exist, create new ones following the same pattern.

For documents — 4 operation-specific policies:

documents_select (SELECT) — USING clause:
```sql
COALESCE(super_admin, 'false') = 'true'
OR scope = 'product'
OR (org_id_match AND archived_at IS NULL)
```

documents_insert (INSERT) — WITH CHECK clause:
```sql
COALESCE(super_admin, 'false') = 'true'
OR (org_id_match AND archived_at IS NULL)
```

documents_update (UPDATE) — USING + WITH CHECK:
```sql
COALESCE(super_admin, 'false') = 'true'
OR (org_id_match AND archived_at IS NULL)
```

documents_delete (DELETE) — USING clause:
```sql
COALESCE(super_admin, 'false') = 'true'
OR (org_id_match AND archived_at IS NULL)
```
