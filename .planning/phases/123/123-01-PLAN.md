But there's a circular problem: useDocsNav can't poll getCacheVersion without a timer or event.

Better approach: expose an event emitter or a React-friendly signal.

RECOMMENDED PATTERN — use a React context counter in ProtectedRoute or useOrgTokenExchange:
The cleanest solution is to add an `orgSwitchCount` counter to a context that increments on each successful token exchange. useDocsNav reads it as a dependency.

Since useOrgTokenExchange is called in ProtectedRoute and onOrgChange fires invalidateDocsCache already, we can co-locate the signal in docs-service.ts using a simple pub/sub with a Set of listeners:

```typescript
// docs-service.ts — add pub/sub:
const _listeners = new Set<() => void>()
export function onDocsCacheInvalidated(cb: () => void): () => void {
  _listeners.add(cb)
  return () => _listeners.delete(cb)
}
export function invalidateDocsCache(): void {
  docsCache = null
  docsCachePromise = null
  _listeners.forEach(cb => cb())
}
```

Then useDocsNav subscribes to this event:
```typescript
useEffect(() => {
  const unsubscribe = onDocsCacheInvalidated(() => {
    setRefetchTick(t => t + 1)
  })
  return unsubscribe
}, [])

useEffect(() => {
  // fetch docs — refetchTick is the dep that triggers this
}, [refetchTick])
```

This is clean, zero extra context, zero polling, and fires immediately on org switch.
