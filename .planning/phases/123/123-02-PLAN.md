<objective>
Fix impersonation mode so admin sees the target org's actual data (docs, modules) instead of their own session data.

Purpose: When an admin impersonates an org, the Supabase token is swapped and docs cache is invalidated — but tenant_modules is NOT reloaded. The sidebar module switcher still shows modules from the admin's own org, not the impersonated org. This plan fixes the module reload on impersonation enter/exit.

Output:
- ImpersonationContext.tsx — triggers module refresh after entering/exiting impersonation
- useModuleEnabled.tsx — exposes a refresh mechanism that re-fetches tenant_modules for the current activeOrg

The module fetch uses `activeOrg.id` from Clerk — but when impersonating, Clerk's active org does NOT change. The Supabase token has a different org_id (the impersonated one), but Clerk's session/org is unchanged.

This means: during impersonation, `activeOrg.id` is still the admin's org. Querying `tenant_modules` with the admin's org_id and the impersonation Supabase token will return different data (the impersonated org's RLS view), but the WHERE clause `eq('org_id', activeOrg.id)` filters by the ADMIN's org_id, not the impersonated org_id.

The fix is NOT to change the WHERE clause — that would be wrong and insecure. Instead, when impersonating, the module fetch should use the IMPERSONATED org_id.

STEP 1 — Create src/platform/module-loader/module-signals.ts:
```typescript
/**
 * Module invalidation pub/sub.
 *
 * Used by ImpersonationContext to notify useModuleEnabled when impersonation
 * starts or ends, so it can re-fetch tenant_modules for the effective org.
 *
 * Also tracks the impersonated org ID so useModuleEnabled knows which org to query.
 */

const _moduleListeners = new Set<() => void>()

/** The org_id to use for tenant_modules queries during impersonation (null = use own org). */
let _impersonatedOrgId: string | null = null

/**
 * Subscribe to module invalidation events.
 * Returns an unsubscribe function.
 */
export function onModulesInvalidated(cb: () => void): () => void {
  _moduleListeners.add(cb)
  return () => { _moduleListeners.delete(cb) }
}

/**
 * Notify all listeners that modules should be re-fetched.
 * Called by ImpersonationContext on enter/exit.
 */
export function invalidateModules(): void {
  _moduleListeners.forEach((cb) => { cb() })
}

/**
 * Set the org_id override for impersonation mode.
 * Pass null to revert to the user's own active org.
 */
export function setImpersonationOrgId(orgId: string | null): void {
  _impersonatedOrgId = orgId
}

/**
 * Get the current impersonated org_id, or null if not impersonating.
 */
export function getImpersonationOrgId(): string | null {
  return _impersonatedOrgId
}
```
