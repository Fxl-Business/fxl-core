  <name>Task 1: Create BrandingConfig type and branding utility library</name>
  <files>tools/wireframe-builder/types/branding.ts, tools/wireframe-builder/lib/branding.ts</files>
  <action>
Create two files:

**types/branding.ts** -- BrandingConfig type and DEFAULT_BRANDING:
- Fields (per user decision): primaryColor (string hex), secondaryColor (string hex), accentColor (string hex), headingFont (string), bodyFont (string), logoUrl (string), faviconUrl (string, optional)
- JSDoc comments on each field explaining what it controls
- DEFAULT_BRANDING constant with neutral gray palette: primaryColor '#374151' (gray-700), secondaryColor '#6B7280' (gray-500), accentColor '#3B82F6' (blue-500), headingFont 'Inter', bodyFont 'Inter', logoUrl '' (empty = FXL text fallback)
- Export both the type and the default constant

**lib/branding.ts** -- Utility functions:
1. `resolveBranding(config?: Partial<BrandingConfig>): BrandingConfig` -- merge partial with DEFAULT_BRANDING
2. `brandingToCssVars(branding: BrandingConfig): React.CSSProperties` -- generate CSS custom properties object:
   - `--brand-primary`, `--brand-secondary`, `--brand-accent`
   - `--brand-primary-light`, `--brand-primary-dark` (derived)
   - `--brand-chart-1` through `--brand-chart-5` (from derivePalette)
   - `--brand-heading-font`, `--brand-body-font`
   - Cast return as `React.CSSProperties` (CSS vars need type assertion)
3. `derivePalette(branding: BrandingConfig)` -- generate 5 chart colors from 3 base:
   - chart1 = primaryColor, chart2 = secondaryColor, chart3 = accentColor
   - chart4 = mix of primary+secondary (50%), chart5 = mix of secondary+accent (50%)
   - primaryLight = lighten primary by 30, primaryDark = darken primary by 20
4. `getFontLinks(branding: BrandingConfig): string[]` -- return Google Fonts CSS URLs for non-Inter fonts
5. Helper functions: `hexToHsl`, `hslToHex`, `lighten`, `darken`, `mixColors` -- ~20 lines each, operating in HSL space

IMPORTANT: Use `--brand-*` prefix for ALL variables (never `--primary` or `--accent` which conflict with app theme from Phase 02.3).

Note on recharts (from research pitfall): CSS var() does NOT work in SVG fill/stroke attributes. The chart components in Plan 02 will receive resolved hex colors as props, NOT via CSS cascade. The CSS vars are for non-SVG elements (sidebar bg, table header bg, font family). Include a `getChartPalette(branding: BrandingConfig): string[]` function that returns an array of 5 resolved hex strings for recharts components to use directly.
