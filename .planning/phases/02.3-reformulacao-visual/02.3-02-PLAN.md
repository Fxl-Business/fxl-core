  <action>
The MarkdownRenderer uses React inline `style={{}}` with hardcoded hex values for code rendering. This is the primary pitfall from RESEARCH.md (Pitfall 1).

1. Find all inline style objects with hardcoded colors (e.g., `background: '#0f172a'`, `color: '#e2e8f0'`, `borderColor: '#1e293b'`).

2. Replace each with Tailwind classes that use semantic tokens or the custom --code-bg/--code-fg variables from Plan 01:
   - `<pre>` background: inline hex -> class `bg-[hsl(var(--code-bg))]` or a dedicated Tailwind class
   - `<code>` text color: inline hex -> class using semantic token
   - Any border colors: inline hex -> `border-border` or appropriate token

3. If any inline styles CANNOT be replaced with Tailwind classes (edge cases), use CSS variables: `style={{ backgroundColor: 'hsl(var(--code-bg))' }}` -- this keeps them theme-aware even if inline.

4. Verify that code blocks, inline code, and pre blocks look correct in both light and dark contexts (they use the --code-bg/--code-fg variables that switch with .dark class).

5. Do NOT change the component's API (props, exports). Only change visual styling.
