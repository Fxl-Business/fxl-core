# Plan 126-02: Refactor Consumers + ImpersonationContext + Tests

## Objective

Migrate all consumers of the old `useOrgTokenExchange()` hook and `setOrgAccessToken()` / `getOrgAccessToken()` imperative API to use the new `useOrgToken()` context hook. Remove or deprecate the old hook. Update `ImpersonationContext` to use context methods instead of direct token mutation. Adapt existing tests and add new tests for AbortController behavior.

Update `src/platform/auth/ImpersonationContext.tsx` to use `useOrgToken()` instead of direct `setOrgAccessToken()` / `getOrgAccessToken()`:

1. Replace imports:
   ```typescript
   // REMOVE: import { setOrgAccessToken, getOrgAccessToken } from '@platform/supabase'
   // ADD:
   import { useOrgToken } from '@platform/tenants/OrgTokenContext'
   ```

2. Inside `ImpersonationProvider`, add:
   ```typescript
   const { setTokenOverride, clearTokenOverride } = useOrgToken()
   ```

3. In `enterImpersonation` callback:
   - REMOVE: `originalTokenRef.current = getOrgAccessToken()`
   - REMOVE: `setOrgAccessToken(result.access_token)`
   - ADD: `setTokenOverride(result.access_token)` (this internally saves the original token)
   - Remove `originalTokenRef` ref entirely — the provider handles save/restore
   - Add `setTokenOverride` to the useCallback dependency array

4. In `exitImpersonation` callback:
   - REMOVE: `setOrgAccessToken(originalTokenRef.current)`
   - REMOVE: `originalTokenRef.current = null`
   - ADD: `clearTokenOverride()` (this internally restores the saved token)
   - Add `clearTokenOverride` to the useCallback dependency array

5. Remove `const originalTokenRef = useRef<string | null>(null)` — no longer needed

The rest of `ImpersonationContext` (impersonation state, module invalidation, setImpersonationOrgId) stays unchanged.
</action>

<read_first>

<action>
Create `src/platform/tenants/OrgTokenContext.test.ts` with tests covering:

1. **Happy path** — token exchange completes, `isReady` becomes true, `orgToken` is set
2. **Error path** — exchange fails, `error` is set, `isReady` stays false
3. **No session** — does not attempt exchange when session is null
4. **No activeOrg** — does not attempt exchange when activeOrg is null
5. **Org switch resets isReady** — when org changes, `isReady` goes false then true after new exchange
6. **Org switch aborts previous controller** — when org changes, `AbortController.abort()` is called before creating new controller. Verify by checking the `signal` passed to `exchangeToken` has `signal.aborted === true` after switch.
7. **AbortError is silenced** — when `exchangeToken` throws `DOMException` with name `AbortError`, `error` remains null (not surfaced)
8. **onOrgChange callback fires on org switch** — callback fires after successful exchange on org change
9. **setTokenOverride / clearTokenOverride** — calling `setTokenOverride('override-token')` changes `orgToken`, calling `clearTokenOverride()` restores original
10. **Throws when used outside provider** — `useOrgToken()` outside `OrgTokenProvider` throws `'useOrgToken must be used inside OrgTokenProvider'`
