The existing path aliases are:
- `@/` -> `src/` (in both tsconfig.json and vite.config.ts)
- `@tools/` -> `tools/`
- `@clients/` -> `clients/`

Since `@/` already maps to `src/`, imports like `@/platform/layout/Layout` and `@/shared/ui/button` will resolve automatically via the existing `@/*` alias. No NEW aliases are strictly needed for the base paths.

However, to make the module boundary intention explicit and enable future tooling (like lint rules that enforce import boundaries), add these OPTIONAL convenience aliases:

**In tsconfig.json `compilerOptions.paths`**, add:
```json
"@platform/*": ["src/platform/*"],
"@shared/*": ["src/shared/*"],
"@modules/*": ["src/modules/*"]
```

These go AFTER the existing aliases. Keep all existing aliases (`@/*`, `@tools/*`, `@clients/*`) untouched.

**In vite.config.ts `resolve.alias`**, add matching entries:
```typescript
'@platform': path.resolve(__dirname, 'src/platform'),
'@shared': path.resolve(__dirname, 'src/shared'),
'@modules': path.resolve(__dirname, 'src/modules'),
```

These go AFTER the existing aliases. Keep all existing aliases (`@`, `@tools`, `@clients`) untouched.

After updating both files, run `npx tsc --noEmit` and `npm run build` to verify nothing is broken. The new aliases should compile cleanly since they point to directories with only .gitkeep files (no TypeScript to parse).
