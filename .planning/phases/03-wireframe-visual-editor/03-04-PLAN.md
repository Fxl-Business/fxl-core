**1. BlueprintRenderer.tsx** -- Major modification

Current: renders flat `screen.sections` array with SectionWrapper for comments.
New: renders grid rows with DndContext for drag-and-drop, editable section wrappers, add section buttons.

New props (backward compatible -- all edit props optional):
```typescript
type Props = {
  screen: BlueprintScreen
  clientSlug?: string
  comments?: Comment[]
  onOpenComments?: (targetId: string, label: string) => void
  // Edit mode props (all optional for backward compat)
  editMode?: boolean
  selectedSection?: { rowIndex: number; cellIndex: number } | null
  onSelectSection?: (rowIndex: number, cellIndex: number) => void
  onDeleteSection?: (rowIndex: number, cellIndex: number) => void
  onAddSection?: (rowIndex: number, section: BlueprintSection) => void
  onReorderRows?: (oldIndex: number, newIndex: number) => void
  onChangeLayout?: (rowIndex: number, layout: GridLayout) => void
  rows?: ScreenRow[]  // when provided, use rows instead of screen.sections
}
```

Implementation:
1. Determine the render source: if `rows` prop is provided, use rows. Otherwise, convert `screen.rows ?? sectionsToRows(screen.sections)` into rows (auto-wrap flat sections).
2. Wrap everything in a DndContext (from @dnd-kit/core) with closestCenter collision detection and PointerSensor + KeyboardSensor. Only active when `editMode` is true.
3. Inside DndContext, use SortableContext with verticalListSortingStrategy. Items are the row IDs.
4. For each row:
   - If editMode, show GridLayoutPicker at the top-left of the row
   - Render the grid using Tailwind grid classes from GRID_LAYOUTS[row.layout].cols
   - For '2-1' layout: first cell gets `col-span-2`, second gets `col-span-1`
   - For '1-2' layout: first cell gets `col-span-1`, second gets `col-span-2`
   - Each cell renders: EditableSectionWrapper (if editMode) > SectionWrapper (if comments) > SectionRenderer
   - If editMode, show AddSectionButton between each row and at the bottom
5. On DragEnd: call `onReorderRows(oldIndex, newIndex)` using the row IDs to map back to indices
6. Keep the WireframeFilterBar rendering at the top (unchanged)

When editMode is false or undefined: renders identically to current behavior (no visual change for read-only mode). The DndContext and SortableContext are still present but inactive (no sensors when !editMode).

**2. SectionWrapper.tsx** -- Minor modification

Currently wraps sections with comment icon/badge. Extend to coexist with EditableSectionWrapper.

No changes needed to the component itself -- it already accepts children and wraps them. The layering in BlueprintRenderer handles the nesting:
- Edit mode ON + comments ON: `EditableSectionWrapper > SectionWrapper > SectionRenderer`
- Edit mode ON + comments OFF: `EditableSectionWrapper > SectionRenderer`
- Edit mode OFF + comments ON: `SectionWrapper > SectionRenderer` (current behavior)
- Edit mode OFF + comments OFF: `SectionRenderer` (current behavior)

Only modify SectionWrapper if needed for TypeScript compatibility. The key requirement is that it continues working unchanged in the new nesting context. Verify it compiles with the new BlueprintRenderer.

IMPORTANT: Preserve backward compatibility. When no edit props are passed (SharedWireframeView case, or any consumer not using edit mode), the renderer MUST behave exactly as before.

**1. WireframeViewer.tsx** -- Major rewrite

Replace the hardcoded blueprint import with Supabase-backed loading. Add full edit mode state management.

Key changes:

**Data loading:**
- Remove `import blueprint from '@clients/financeiro-conta-azul/wireframe/blueprint.config'`
- Add `import seedConfig from '@clients/financeiro-conta-azul/wireframe/blueprint.config'` (keep the import but rename -- used for seeding only)
- On mount (useEffect), call `loadBlueprint('financeiro-conta-azul')`. If null, call `seedFromFile('financeiro-conta-azul', seedConfig, user?.id ?? 'system')` then `loadBlueprint` again.
- Store loaded config in `const [config, setConfig] = useState<BlueprintConfig | null>(null)`
- Show loading spinner while config is null
- Derive `screens` from `config.screens` (not from the imported file)

**Edit mode state:**
```typescript
const [editMode, setEditMode] = useState<EditModeState>({
  active: false, dirty: false, saving: false, selectedSection: null
})
const [workingConfig, setWorkingConfig] = useState<BlueprintConfig | null>(null)
```

- When entering edit mode: `setWorkingConfig(structuredClone(config))` -- deep copy
- When exiting edit mode: if dirty, show confirmation dialog ("Voce tem alteracoes nao salvas. Deseja sair sem salvar?"). If confirmed or not dirty: `setWorkingConfig(null)`, reset editMode.
- When saving: set `saving: true`, call `saveBlueprint(slug, workingConfig, user.id)`, on success set `config = workingConfig`, `dirty: false`, `saving: false`. On error: show error (console.error + keep saving false), leave dirty true.

**Derive active screen from working copy when editing:**
```typescript
const activeConfig = editMode.active && workingConfig ? workingConfig : config
const screens = activeConfig?.screens ?? []
const activeScreen = screens[activeIndex]
```

**Rows management:**
- Convert screen sections to rows on load: `const rows = activeScreen.rows ?? sectionsToRows(activeScreen.sections)`
- When working config changes (edit mode), update rows in the working config
- All edit operations update `workingConfig` and set `dirty: true`

**Edit operation handlers:**
- `handleSelectSection(rowIndex, cellIndex)`: set editMode.selectedSection
- `handleDeleteSection(rowIndex, cellIndex)`: remove section from row, if row becomes empty remove the row. Update workingConfig + dirty.
- `handleAddSection(rowIndex, section)`: add a new row after rowIndex with layout '1' and the section. Update workingConfig + dirty.
- `handleReorderRows(oldIndex, newIndex)`: use arrayMove on the rows array. Update workingConfig + dirty.
- `handleChangeLayout(rowIndex, layout)`: update row.layout. If new layout has fewer cells, move excess sections to new rows below. Update workingConfig + dirty.
- `handlePropertyChange(updatedSection)`: replace the section at editMode.selectedSection position in workingConfig. Set dirty.
- `handleAddScreen(screen)`: add screen to workingConfig.screens. Set dirty.
- `handleDeleteScreen(index)`: remove screen from workingConfig.screens. If active was deleted, set activeIndex to 0. Set dirty.
- `handleRenameScreen(index, title)`: update screen title. Set dirty.
- `handleReorderScreens(screens)`: replace workingConfig.screens. Set dirty.

**Screen switch with dirty check:**
When activeIndex changes and editMode.dirty is true: show confirmation before switching. Implement via a pendingScreenSwitch state.

**Layout restructure:**
Replace the inline sidebar with ScreenManager component. Add AdminToolbar above the main content area. Add PropertyPanel.
