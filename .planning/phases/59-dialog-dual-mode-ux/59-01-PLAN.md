  <action>
    Create a preview card component that renders a scaled-down mini-preview of a section type.

    **Props:**
    ```typescript
    type Props = {
      type: BlueprintSection['type']
      onSelect: (type: BlueprintSection['type']) => void
    }
    ```

    **Implementation details:**

    1. **Read from registry:** Get `SECTION_REGISTRY[type]` to access `defaultProps()`, `catalogEntry` (label, description, icon, category).

    2. **Mini-preview container:** A fixed-height card (~160-180px) with:
       - A preview area (top ~120-140px) that contains the actual section render
       - A label bar (bottom ~30-40px) showing icon + label

    3. **Preview rendering:** Inside the preview area:
       - Wrap in `<WireframeThemeProvider externalTheme="light">` to ensure consistent theme
       - Use CSS `transform: scale(0.25)` with `transform-origin: top left` and a large virtual container (e.g., `width: 400%`, `height: 400%`) to show the section at 25% scale
       - Set `overflow: hidden` on the outer container to clip the scaled content
       - Set `pointer-events: none` on the preview area to prevent any interactive elements in the rendered section from capturing clicks
       - Render using: `<SectionRenderer section={SECTION_REGISTRY[type].defaultProps()} compareMode={false} comparePeriod="" />`

    4. **Label bar:** Below the preview area:
       - Show the catalogEntry icon (16x16) + label text (text-xs, font-medium)
       - Truncate long labels with `truncate` class

    5. **Click behavior:**
       - The entire card is a `<button>` element
       - On click: call `onSelect(type)` — NO confirmation, no intermediate step
       - Hover state: `hover:border-primary hover:shadow-sm` transition
       - Cursor: `cursor-pointer`

    6. **Card styling:**
       - `rounded-lg border bg-card` base
       - `overflow-hidden` to clip preview content
       - The preview area background should be a neutral light gray (`bg-muted/30`) to provide contrast
       - Clean, professional appearance matching shadcn/ui aesthetics

    7. **Error boundary:** Wrap the SectionRenderer call in a try-catch or React ErrorBoundary pattern. If a section fails to render (unlikely but possible), show a fallback with just the icon and label. Simplest approach: use a small wrapper component with `componentDidCatch`-style error handling, OR just use a simple conditional — if the registry entry doesn't exist, show a placeholder.

    **Important:** Do NOT call `defaultProps()` on every render — use `useMemo` to memoize the generated section props per type, since `defaultProps()` creates a new object each time.
  </action>
