# Plan 88-01: TypeScript Zero-Errors Verification

## Objective

Verify that `npx tsc --noEmit` produces zero errors across the entire codebase. If errors exist, fix them. Document the result in an audit report.

## Tasks

### Task 1: Run Full TypeScript Check

<read_first>
- tsconfig.json (to understand compiler options and included paths)
- CLAUDE.md (project rules, especially "never use any")
</read_first>

<action>
Run `npx tsc --noEmit` and capture the output.

If zero errors: record "PASS" in the audit report.

If errors found:
1. Categorize each error by file and type (missing import, type mismatch, missing property, etc.)
2. Fix each error following these rules:
   - NEVER use `any` -- use proper types, generics, or specific type assertions
   - If errors are in Phase 85/86/87 code, fix in place
   - If errors are in older code, fix with minimal changes to avoid regressions
   - Prioritize: type narrowing, proper interface definitions, missing imports
3. After fixing, run `npx tsc --noEmit` again to confirm zero errors
4. If new errors appear from fixes, iterate until clean

Document all fixes made (file, error, fix applied) in the audit report.
</action>

<acceptance_criteria>
- `npx tsc --noEmit` exits with code 0 (zero errors)
- No `any` type exists in any fix applied
- File `.planning/phases/88-quality-gate-security-audit/88-AUDIT-REPORT.md` exists and contains a "## TypeScript Verification" section with either "PASS — zero errors" or a list of errors fixed
</acceptance_criteria>
