Create migration file `supabase/migrations/010_platform_settings.sql` with:

1. Create table `platform_settings`:
```sql
CREATE TABLE IF NOT EXISTS public.platform_settings (
  key         text        PRIMARY KEY,
  value       text        NOT NULL DEFAULT '',
  description text        NOT NULL DEFAULT '',
  updated_at  timestamptz NOT NULL DEFAULT now()
);
```

2. Enable RLS:
```sql
ALTER TABLE public.platform_settings ENABLE ROW LEVEL SECURITY;
```

3. Create RLS policy — super_admin only for write, all authenticated can read:
```sql
-- Read: any authenticated user can read settings (feature flags need to be readable)
CREATE POLICY "platform_settings_read" ON public.platform_settings
  FOR SELECT TO anon, authenticated
  USING (true);

-- Write (insert/update/delete): only super_admin
CREATE POLICY "platform_settings_write" ON public.platform_settings
  FOR ALL TO authenticated
  USING (
    COALESCE(
      nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'super_admin',
      'false'
    ) = 'true'
  )
  WITH CHECK (
    COALESCE(
      nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'super_admin',
      'false'
    ) = 'true'
  );
```

4. Seed initial feature flag rows:
```sql
INSERT INTO public.platform_settings (key, value, description) VALUES
  ('feature.tenant_self_service', 'false', 'Allow tenants to self-register'),
  ('feature.module_marketplace', 'false', 'Enable module marketplace for tenants'),
  ('platform.maintenance_mode', 'false', 'Put platform in maintenance mode'),
  ('platform.max_tenants', '100', 'Maximum number of tenants allowed')
ON CONFLICT (key) DO NOTHING;
```

Note: This follows the same COALESCE pattern from migration 008, but checks `super_admin` claim instead of `org_id`. The read policy is open to all because feature flags need to be readable by the app.
