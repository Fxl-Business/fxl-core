# Plan 128-02: Integrate Retry into Critical Services

<objective>
Wrap all critical fetch calls in token exchange hook, admin-service, and tenant-service with the `withRetry` utility from Plan 01. Token exchange gets retry in the hook (not in the pure function). Admin and tenant services wrap each fetch function internally. All use the same utility â€” zero logic duplication.
</objective>

- src/platform/tenants/token-exchange.ts (exchangeToken stays unchanged â€” verify it's still pure)

<action>
Modify `src/platform/tenants/useOrgTokenExchange.ts`:

1. Add import at top:
```typescript
import { withRetry } from '@platform/lib/retry'
```

2. In the `doExchange()` function inside the useEffect, wrap the `exchangeToken` call with `withRetry`. Replace the line:
```typescript
const result = await exchangeToken(clerkToken, activeOrg!.id)
```
with:
```typescript
const result = await withRetry(
  () => exchangeToken(clerkToken, activeOrg!.id),
  { maxRetries: 3, baseDelay: 1000, backoffFactor: 2 },
)
```

**Important:** Do NOT modify `token-exchange.ts`. The retry wraps the call in the hook, keeping `exchangeToken` as a pure single-fetch function.

The existing `catch` block in `doExchange()` already handles the final error with `setError(message)` and `console.error`. The `withRetry` will log `console.warn` for each retry attempt, and only throw after all retries are exhausted, at which point the existing catch block handles it.
</action>

<action>
Modify `src/platform/services/admin-service.ts`:

1. Add import at top:
```typescript
import { withRetry } from '@platform/lib/retry'
```

2. Wrap each of the 5 exported async functions with `withRetry`. The pattern is to wrap the entire fetch+response-handling block.

For each function, wrap the body in withRetry. Example for `listUsers`:

**Before:**
```typescript
export async function listUsers(): Promise<AdminUserListResponse> {
  const res = await fetch(`${FUNCTIONS_URL}/admin-users`, {
    headers: await getAuthHeaders(),
  })
  if (!res.ok) throw new Error(`Failed to list users: ${res.status}`)
  return res.json() as Promise<AdminUserListResponse>
}
```

**After:**
```typescript
export async function listUsers(): Promise<AdminUserListResponse> {
  return withRetry(async () => {
    const res = await fetch(`${FUNCTIONS_URL}/admin-users`, {
      headers: await getAuthHeaders(),
    })
    if (!res.ok) throw new Error(`Failed to list users: ${res.status}`)
    return res.json() as Promise<AdminUserListResponse>
  })
}
```

Apply the same pattern to all 5 functions:
- `listUsers` â€” wrap entire body
- `listOrgMembers(orgId)` â€” wrap entire body (captures orgId in closure)
- `addOrgMember(orgId, userId)` â€” wrap entire body
- `removeOrgMember(orgId, userId)` â€” wrap entire body
- `getImpersonationToken(orgId)` â€” wrap entire body

All use default retry options (no explicit options object needed â€” defaults are maxRetries:3, baseDelay:1000, backoffFactor:2).

**Do NOT modify:** `setAdminClerkTokenGetter`, `getAuthHeaders`, type re-exports. These stay unchanged.
</action>
