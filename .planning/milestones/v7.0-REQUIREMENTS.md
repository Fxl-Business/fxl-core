# Requirements: Nexo

**Defined:** 2026-03-18
**Core Value:** Nexo e o hub central multi-tenant — cada empresa ve tudo sobre si mesma

## v7.0 Requirements

Requirements for Admin-Only Org Management milestone.

### Access Control

- [x] **ACC-01**: Self-service org creation (`/criar-empresa`) is removed — no client-side `createOrganization()` calls
- [x] **ACC-02**: Only super admin can create organizations (via admin panel)
- [x] **ACC-03**: User without org membership sees `/solicitar-acesso` screen with clear instructions
- [x] **ACC-04**: `/solicitar-acesso` screen has Nexo branding, message explaining how to request access, and sign out button

### Admin User Management

- [x] **ADM-01**: Admin can view list of unaffiliated users (users with 0 org memberships) in admin panel
- [x] **ADM-02**: Admin can filter users by status (all / unaffiliated / affiliated)
- [x] **ADM-03**: Admin can link an unaffiliated user to an organization directly from users list
- [x] **ADM-04**: Admin can search existing unaffiliated users when adding members to a tenant

### Tenant Archival

- [x] **ARC-01**: Admin can archive a tenant via soft-delete (sets `archived_at` timestamp on all org-scoped data)
- [x] **ARC-02**: Archiving a tenant sets Clerk org metadata `archived=true` and removes all org memberships
- [x] **ARC-03**: Archived tenant data is hidden from normal queries (RLS filters `archived_at IS NULL`)
- [x] **ARC-04**: Admin can view archived tenants in a separate tab on tenants page
- [x] **ARC-05**: Admin can restore an archived tenant (reverses soft-delete, restores Clerk org metadata)

### Admin Dashboard

- [x] **DASH-01**: Dashboard shows count of unaffiliated users
- [x] **DASH-02**: Dashboard shows count of archived tenants
- [x] **DASH-03**: Dashboard has quick action links to unaffiliated users and archived tenants views

## Future Requirements

### Invitation System

- **INV-01**: Admin can send email invitations to users for a specific org
- **INV-02**: User can accept invitation via email link and join org automatically

### Access Request Workflow

- **REQ-01**: User can submit access request from `/solicitar-acesso` screen
- **REQ-02**: Admin sees pending access requests in admin panel
- **REQ-03**: Admin can approve/deny access requests

## Out of Scope

| Feature | Reason |
|---------|--------|
| Email invitations | Future milestone — manual process via admin panel for now |
| Access request form/workflow | Future milestone — admin checks unaffiliated users manually |
| Self-service org creation behind feature flag | Removed entirely, not gated |
| Hard delete of tenant data | Always soft-delete for compliance/recovery |
| Tenant data export before archival | Future — archive preserves data in DB |
