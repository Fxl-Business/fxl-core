```typescript
import { useMemo, useState } from 'react'
import { MODULE_REGISTRY } from '@platform/module-loader/registry'
import { buildGraph } from './diagram-utils'
import type { GraphNode, GraphEdge } from './diagram-types'
```

**Component structure:**

1. `ModuleDiagram` (default export) — top-level component:
   - Calls `const { nodes, edges } = useMemo(() => buildGraph(MODULE_REGISTRY), [])` — MODULE_REGISTRY is static, so empty deps array is correct
   - Manages `const [hoveredNodeId, setHoveredNodeId] = useState<string | null>(null)`
   - Renders a responsive container `<div>` with `<svg>` using `viewBox="0 0 600 300"` and `className="w-full h-auto"`
   - SVG contains: arrow marker `<defs>`, edges as `<DiagramEdge>`, nodes as `<DiagramNode>`

2. `DiagramNode` — renders a single module node as SVG group:
   - Props: `node: GraphNode`, `isHovered: boolean`, `isDimmed: boolean`, `onMouseEnter: () => void`, `onMouseLeave: () => void`
   - Renders `<g>` containing:
     - `<rect>` — 160x56 rounded rectangle (rx=12). Fill and stroke vary by mode:
       - Default: `fill="white" stroke="#e2e8f0"` with `class="dark:fill-slate-800 dark:stroke-slate-700"`
       - Hovered: `stroke="#6366f1"` (indigo-500), `strokeWidth={2}`
       - Dimmed: `opacity={0.35}`
     - `<text>` — module label, centered horizontally, at y offset for vertical centering. Font: 13px semibold, `fill="#1e293b"` / `dark:fill-white`
     - `<rect>` — small status badge below label. Colors from STATUS_CLASSES pattern:
       - active: emerald fill
       - beta: amber fill
       - coming-soon: slate fill
     - `<text>` — status label text inside badge rect. Use STATUS_LABELS: `{ active: 'Ativo', beta: 'Beta', 'coming-soon': 'Em breve' }`
   - Mouse events: `onMouseEnter` sets hoveredNodeId, `onMouseLeave` clears it
   - Add `data-module-id={node.id}` on the `<g>` for Phase 142 click-to-scroll integration
   - Add `cursor: pointer` style for future click interaction

3. `DiagramEdge` — renders a single edge as SVG path:
   - Props: `edge: GraphEdge`, `sourceNode: GraphNode`, `targetNode: GraphNode`, `isHighlighted: boolean`, `isDimmed: boolean`
   - Renders `<line>` or `<path>` from source center to target center
   - Stroke color:
     - Default: `#94a3b8` (slate-400), dark mode `#64748b` (slate-500)
     - Highlighted: `#6366f1` (indigo-500), strokeWidth 2
     - Dimmed: opacity 0.15
   - Arrow marker via `marker-end="url(#arrowhead)"`
   - Transition: `transition: opacity 200ms, stroke 200ms` for smooth hover effect

4. SVG `<defs>` section for arrowhead marker:
   ```xml
   <defs>
     <marker id="arrowhead" markerWidth="8" markerHeight="6" refX="8" refY="3" orient="auto">
       <polygon points="0 0, 8 3, 0 6" className="fill-slate-400 dark:fill-slate-500" />
     </marker>
     <marker id="arrowhead-highlighted" markerWidth="8" markerHeight="6" refX="8" refY="3" orient="auto">
       <polygon points="0 0, 8 3, 0 6" className="fill-indigo-500" />
     </marker>
   </defs>
   ```

**Hover logic in ModuleDiagram:**
- When `hoveredNodeId` is set, compute `connectedNodeIds` as the set of nodes connected to hoveredNodeId via any edge (source or target)
- For each node: `isHovered = node.id === hoveredNodeId`, `isDimmed = hoveredNodeId !== null && node.id !== hoveredNodeId && !connectedNodeIds.has(node.id)`
- For each edge: `isHighlighted = hoveredNodeId !== null && (edge.source === hoveredNodeId || edge.target === hoveredNodeId)`, `isDimmed = hoveredNodeId !== null && !isHighlighted`

<action>
Ensure the ModuleDiagram component handles both color modes correctly.

SVG elements do not support Tailwind `dark:` variants natively. The component must detect the current color mode and apply appropriate colors.

Approach: Use a simple hook or inline check. The Nexo app uses Tailwind dark mode with `class` strategy (the `<html>` element has `class="dark"` when dark mode is active).

Add dark mode detection to ModuleDiagram:

```typescript
function useDarkMode(): boolean {
  const [isDark, setIsDark] = useState(() =>
    document.documentElement.classList.contains('dark')
  )

  useEffect(() => {
    const observer = new MutationObserver(() => {
      setIsDark(document.documentElement.classList.contains('dark'))
    })
    observer.observe(document.documentElement, { attributes: true, attributeFilter: ['class'] })
    return () => observer.disconnect()
  }, [])

  return isDark
}
```

Then use `isDark` to select fill/stroke colors for all SVG elements:
- Node rect fill: `isDark ? '#1e293b' : '#ffffff'`
- Node rect stroke: `isDark ? '#334155' : '#e2e8f0'`
- Node label fill: `isDark ? '#f8fafc' : '#1e293b'`
- Edge stroke: `isDark ? '#64748b' : '#94a3b8'`
- Highlighted edge: `'#6366f1'` (same in both modes)
- Arrowhead fill: same as edge stroke

Verify that the SVG `<defs>` markers also use the correct colors based on isDark.

The container `<div>` wrapping the SVG CAN use Tailwind dark: variants since it's a regular HTML element.
</action>
