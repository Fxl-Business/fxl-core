   import * as Sentry from '@sentry/react'

   const sentryDsn = import.meta.env.VITE_SENTRY_DSN
   if (sentryDsn) {
     Sentry.init({
       dsn: sentryDsn,
       environment: import.meta.env.VITE_SENTRY_ENVIRONMENT ?? 'production',
       integrations: [
         Sentry.browserTracingIntegration(),
         Sentry.replayIntegration({ maskAllText: false, blockAllMedia: false }),

       replaysSessionSampleRate: 0,
       replaysOnErrorSampleRate: 1.0,
     })
   }

2. **Extracao de Dados via Connector Module** — How spoke data flows to Hub:
   - Spoke exposes FXL Contract API endpoints (manifest, entities, widgets)
   - Hub Connector module fetches and renders data
   - ConnectorResult<T> pattern for typed responses:
   ```typescript
   // Spoke exposes widgets via GET /api/fxl/widgets/:id/data
   // Response follows FxlKpiData | FxlChartData | FxlTableData | FxlListData

   interface FxlKpiData {
     value: number | string
     label: string
     trend?: number
     prefix?: string
     suffix?: string
   }

   interface FxlChartData {
     labels: string[]
     datasets: { label: string; data: number[] }[]
   }
   ```
   - Callout: "O Connector module do Hub faz fetch com timeout de 5s e retry automatico. O spoke so precisa expor os endpoints corretamente."

3. **Dashboards Padrao para Spoke BI** — What a standard spoke BI dashboard includes:
   - KPI cards: 4-6 metricas principais no topo
   - Trend chart: Evolucao temporal da metrica principal
   - Breakdown table: Detalhamento por dimensao (cliente, produto, regiao)
   - Distribution chart: Composicao (pie/bar) por categoria
   - Reference: "Ver secao Wireframe Builder para templates de dashboard BI."

4. **Widgets do Contract** — Table of 4 widget types:
   | Tipo | Interface | Uso |
   | kpi | FxlKpiData | Metrica numerica com trend |
   | chart | FxlChartData | Grafico de barras/linhas |
   | table | FxlTableData | Tabela com colunas dinamicas |
   | list | FxlListData | Lista de itens com titulo/subtitulo |

5. **Event Tracking** — Lightweight recommendation:
   - Use Vercel Analytics (built-in, zero config) for page views
   - For custom events: simple logging to Supabase table or Sentry breadcrumbs
   - Callout: "Nao instalar SDKs pesados de analytics (Amplitude, Mixpanel) sem aprovacao. Vercel Analytics + Sentry cobrem a maioria dos casos para spokes FXL."
