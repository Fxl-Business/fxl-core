1. **# Banco de Dados** — Opening paragraph: Supabase is the standard database for all FXL spoke projects. This page covers modeling, migrations, RLS, and multi-tenant patterns.

2. **## Modelagem Supabase** — Table design conventions:
   - Every table uses `id uuid DEFAULT gen_random_uuid() PRIMARY KEY`
   - Timestamps: `created_at timestamptz NOT NULL DEFAULT now()`
   - Text columns use `text` (not `varchar`) — Supabase/Postgres best practice
   - Every tenant-scoped table has `org_id text NOT NULL` column
   - Foreign keys use `REFERENCES public.{table}(id)` with explicit schema
   - Example: show the `documents` table structure from migration 007 (title, slug, body, parent_path, scope, sort_order, org_id, created_at, updated_at)

3. **## Migrations Numeradas** — Sequential migration convention:
   - File naming: `NNN_descriptive_name.sql` (3-digit zero-padded, snake_case description)
   - Directory: `supabase/migrations/`
   - All DDL is idempotent: `CREATE TABLE IF NOT EXISTS`, `ADD COLUMN IF NOT EXISTS`, `CREATE INDEX IF NOT EXISTS`
   - Never alter existing migrations — create a new one
   - Show the Nexo migration list (001 through 020) as an example of proper naming
   - Include callout: "Cada migration e um arquivo SQL executado uma unica vez em ordem numerica. Nunca edite uma migration existente — crie uma nova."

4. **## Row Level Security (RLS)** — Three-tier RLS pattern:
   - **Tier 1 — Basic org_id filtering (migration 008):**
     Show the COALESCE pattern: `org_id = COALESCE(nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'org_id', org_id)`
     Explain: when JWT has org_id claim, filter by it; when no JWT (anon), allow all (backward compat)
   - **Tier 2 — Super admin bypass (migration 009):**
     Show the super_admin check prepended: `COALESCE(nullif(...)::jsonb->>'super_admin', 'false') = 'true' OR org_id = ...`
     Explain: super_admin sees all rows across all orgs
   - **Tier 3 — Scope-based access (migration 020):**
     Show the documents-specific policy: product docs readable by all authenticated, tenant docs filtered by org_id
     Explain: different visibility rules for different row types
   - Add cross-reference: "Para detalhes do fluxo de autenticacao e como o JWT chega ao Supabase, veja [Seguranca](/sdk/security)."
   - Include the `ENABLE ROW LEVEL SECURITY` reminder on every new table

5. **## Indices** — Index creation patterns:
   - Naming convention: `idx_{table}_{column}` or `idx_{table}_{column1}_{column2}`
   - Always idempotent: `CREATE INDEX IF NOT EXISTS`
   - Every `org_id` column gets an index (required for RLS performance)
   - Composite indexes for common query patterns (e.g., `(org_id, slug)` for documents)
   - Show real index examples from migration 008 (all 8 org_id indexes)

6. **## Patterns Multi-Tenant** — Multi-tenant architecture patterns:
   - **Adding org_id to existing tables:** Show the migration 008 pattern (ADD COLUMN with default, backfill, then index)
   - **Tenant modules pattern:** Show `tenant_modules` table (org_id + module_id composite PK, config jsonb)
   - **Soft-delete/archival:** Show `archived_at timestamptz` column pattern from migration 019
   - **RLS exclusion of archived data:** `archived_at IS NULL` in RLS policies
   - **Platform vs tenant data:** `scope` column pattern for data that can be 'product' (shared) or 'tenant' (isolated)
   - Include a generic template:
     ```sql
     -- Template: Nova tabela multi-tenant
     CREATE TABLE IF NOT EXISTS public.{nome} (
       id         uuid        PRIMARY KEY DEFAULT gen_random_uuid(),
       org_id     text        NOT NULL,
       -- columns aqui
       created_at timestamptz NOT NULL DEFAULT now(),
       updated_at timestamptz NOT NULL DEFAULT now()
     );
     ALTER TABLE public.{nome} ENABLE ROW LEVEL SECURITY;
     CREATE INDEX IF NOT EXISTS idx_{nome}_org_id ON public.{nome} (org_id);
     -- RLS policy (usar Tier 2 como padrao)
     CREATE POLICY "{nome}_org_access" ON public.{nome}
       FOR ALL TO authenticated
       USING (
         COALESCE(nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'super_admin', 'false') = 'true'
         OR org_id = (nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'org_id')
       )
       WITH CHECK (
         COALESCE(nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'super_admin', 'false') = 'true'
         OR org_id = (nullif(current_setting('request.jwt.claims', true), '')::jsonb->>'org_id')
       );
     ```
