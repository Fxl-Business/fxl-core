Estrutura do conteudo (todas as secoes obrigatorias):

**1. Introducao** — Breve paragrafo explicando que estas sao as regras obrigatorias para todo projeto spoke FXL.

**2. Naming Conventions** — Tabela expandida com exemplos concretos para cada tipo:
- Componentes: PascalCase (`ReservationCard.tsx`)
- Hooks: camelCase com `use` (`useReservations.ts`)
- Services: kebab-case com `-service` (`reservation-service.ts`)
- Utils: camelCase (`formatCurrency.ts`)
- Types/Interfaces: PascalCase (`Reservation`, `ReservationProps`)
- Constantes: UPPER_SNAKE_CASE (`MAX_PAGE_SIZE`)
- Tabelas/colunas DB: snake_case (`reservations`, `check_in_date`)
- Rotas API: kebab-case (`/api/fxl/entities`)
- Callout referenciando `estrutura-projeto.md` para arvore de diretorios

**3. TypeScript Strict Patterns** — Subsecoes com code blocks:
- `strict: true` obrigatorio no tsconfig.json
- Zero `any` — exemplos de como tipar corretamente (generics, union types, unknown + type guard)
- `import type` para importacoes apenas de tipo — exemplo mostrando `import type { Reservation } from '@/types/reservation'`
- Discriminated unions — exemplo com tipo de secao de wireframe ou similar
- Zod validation — exemplo de schema com `z.object()` e `z.infer<typeof schema>`
- Retorno de funcoes tipado explicitamente — exemplo de service function
- Callout: "Nunca usar `as` para silenciar erros — corrigir o tipo"

**4. ESLint Configuration** — Subsecoes:
- ESLint v9 flat config (`eslint.config.js`) — exemplo de config base
- Regras FXL obrigatorias em tabela: `no-explicit-any` (error), `@typescript-eslint/no-unused-vars` (error), `import/order` (warn), `no-console` (warn em prod)
- Boundaries plugin para isolamento de modulos — exemplo de rule config
- Como rodar: `npx eslint .` e integracao com CI

**5. Import Ordering** — Regras com exemplo de arquivo completo:
- Grupo 1: React (`import { useState } from 'react'`)
- Grupo 2: Libs externas (`import { useAuth } from '@clerk/react'`)
- Grupo 3: Path aliases internos (`import { Button } from '@shared/ui/button'`)
- Grupo 4: Imports relativos (`import { ReservationCard } from './ReservationCard'`)
- Grupo 5: Type imports (`import type { Reservation } from '@/types/reservation'`)
- Linha em branco entre grupos
- ESLint rule config para `import/order`

**6. Component Patterns** — Com code blocks mostrando padrao completo:
- Functional components apenas (sem class components)
- Named exports (nunca default export)
- Props interface nomeada `{Component}Props`
- Exemplo completo de componente com Props, destructuring, early returns, e JSX
- Composicao via children e render props — quando usar cada
- Callout: "Hooks de uso unico ficam no mesmo arquivo do componente"

**7. Hook Patterns** — Com code blocks:
- Prefixo `use` obrigatorio
- Retorno tipado com interface ou type nomeado
- Separacao: hooks de data fetching (useReservations) vs hooks de UI state (useToggle)
- Exemplo de hook de data fetching com loading/error/data states
- Exemplo de hook de UI state simples
- Regra: hooks compartilhados em `src/hooks/`, hooks de uso unico junto ao componente

**8. Service Patterns** — Com code blocks:
- Sufixo `-service` no nome do arquivo
- Funcoes puras exportadas (sem classe)
- Tipagem forte de retorno
- Pattern `ConnectorResult<T>` para APIs externas — exemplo com `{ data: T | null, error: string | null }`
- `withRetry` wrapper para chamadas criticas — exemplo de uso
- Callout: "Nunca acessar campos de resposta de API sem fallback (`?? []`, `?? 0`)"

**9. Commit Conventions** — Tabela com exemplos:
- Formato: `tipo(escopo): mensagem`
- Tipos: docs, app, infra, tool, [client-slug]
- Exemplos reais de commits

**10. Quality Gates** — Comandos obrigatorios:
- `npx tsc --noEmit` — zero erros TypeScript
- `npx eslint .` — zero erros ESLint
- `fxl-doctor.sh` — health check completo
