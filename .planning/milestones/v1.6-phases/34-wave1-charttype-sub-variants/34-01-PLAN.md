<objective>
Extend the ChartType union and Zod schema with all 7 new sub-variant literals, then implement the first 4 chart components (Grouped Bar, Step Line, Lollipop, Polar) with full ChartRenderer wiring and editor form entries.

All 7 literals are added upfront (in one atomic schema edit) so that Plan 02 only needs to create leaf components and ChartRenderer cases — no schema changes. This prevents partial compilation issues between plans.
</objective>

<context>
From Phase 34 RESEARCH.md:

**Extension Point A pattern (4-point sync per chart):**
1. `ChartType` union in types/blueprint.ts
2. `chartType` z.enum in blueprint-schema.ts
3. `case` in ChartRenderer.tsx inner switch
4. New leaf component file

**Current state:** 13 chartType values, 10 explicit cases in ChartRenderer, 13 SelectItem entries in BarLineChartForm.

**After this plan:** 20 chartType values (all 7 added to union/enum), 14 explicit cases in ChartRenderer (4 new), 20 SelectItem entries in BarLineChartForm (all 7 added). Plan 02 will add the remaining 3 cases + 3 components.

**Key patterns from existing components:**
- Card wrapper: `rounded-xl border border-wf-card-border bg-wf-card p-4 shadow-sm`
- Title: `text-sm font-bold text-wf-heading`
- Palette: `chartColors ?? ['var(--wf-chart-1)', ...]`
- Axes: `tick={{ fill: 'var(--wf-muted)', fontSize: 11 }}`, `tickLine={false}`, `axisLine={false}`
- Custom legend for multi-series (not Recharts `<Legend>` which cannot resolve CSS vars)

**Mandatory per XCUT requirements:**
- `isAnimationActive={false}` on every Recharts series element (XCUT-02)
- `chartColors` prop accepted and used for fills/legend swatches (XCUT-01)
- `--wf-*` CSS tokens for card/border/text (XCUT-03)

**ChartRenderer default: pitfall:** Unrecognized chartType silently renders as bar chart. Must verify each new chart visually.
</context>
