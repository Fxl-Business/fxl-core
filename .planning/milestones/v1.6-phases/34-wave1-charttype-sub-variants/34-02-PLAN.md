<objective>
Implement the remaining 3 chart sub-variants (Bullet Chart, Range Bar, Bump Chart) and wire them into ChartRenderer, completing all 7 chartType values for Phase 34.

These 3 are the more complex implementations: Bullet uses ReferenceLine with horizontal layout, Range Bar uses CSS-flex (no Recharts), and Bump uses reversed Y-axis with custom end-of-line labels.
</objective>

**Key implementation notes from research:**

**Bullet Chart:**
- Recharts `BarChart layout="vertical"` with single `<Bar>` + `<ReferenceLine>` for target marker
- No native bullet chart in Recharts — composed from primitives
- Mock data: multiple metric rows, each with actual + target value
- `isAnimationActive={false}` on `<Bar>` (XCUT-02)
- Reference bands deferred to ENH-01 (v2)

**Range Bar (CSS-Flex, no Recharts):**
- Pure HTML/CSS flex rows — decision from STATE.md: CSS-flex approach, not Recharts stacked-bar workaround
- Each row: label + bar container with `marginLeft: (start/max)*100%` and `width: ((end-start)/max)*100%`
- Follows CompositionBar pattern (pure HTML/CSS, no Recharts dependency)
- Color per row from `chartColors[i % n]`
- XCUT-02 (isAnimationActive) not applicable — no Recharts elements

**Bump Chart:**
- Recharts `LineChart` with multiple `<Line>` elements + `<YAxis reversed>`
- `domain={[1, entityCount]}` for rank positions, `tickCount={entityCount}` for integer ticks
- Custom `dot` render prop on each Line for end-of-line entity name labels
- `type="monotone"` for smooth rank transitions
- Mock data: 4 entities ranked across 6 periods with rank crossings
- `isAnimationActive={false}` on every `<Line>` (XCUT-02)
</context>

- [ ] Task 1: Create 3 leaf chart components (BulletChart, RangeBar, BumpChart)
  - **BulletChartComponent.tsx** — Horizontal bars with target marker lines
    - Props: `{ title: string, height?: number, categories?: string[], chartColors?: string[] }`
    - Mock data: array of `{ label, actual, target }` entries (e.g., Vendas: 78K actual vs 90K target, Margem: 32% vs 40%)
    - Implementation: `BarChart layout="vertical"` with single `<Bar dataKey="actual" isAnimationActive={false}>` and a `<ReferenceLine>` segment per target value
    - Since Recharts `ReferenceLine` works on the numeric axis: use `x={target}` on a vertical-layout BarChart (numeric axis is X)
    - Target marker styling: `stroke={palette[1]}`, `strokeWidth={2}`, `strokeDasharray="4 4"`
    - `<YAxis type="category" dataKey="label" width={80}>` for metric labels
    - `<XAxis type="number">` for value scale
    - Custom legend showing "Realizado" (palette[0]) and "Meta" (palette[1]) indicators
    - Height default: 200 (compact horizontal display)
  - **RangeBarChartComponent.tsx** — Pure CSS-flex rows (no Recharts)
    - Props: `{ title: string, height?: number, categories?: string[], chartColors?: string[] }`
    - Mock data: 5-6 items with `{ label, start, end }` (e.g., Projeto A: start 2, end 7 on a 0-12 scale)
    - Implementation: for each item, render a row with:
      - Left label div with `text-wf-muted text-xs`
      - Flex bar container (relative, full width, h-6, bg-wf-bg, rounded)
      - Inner bar div positioned with `style={{ marginLeft: (start/max)*100 + '%', width: ((end-start)/max)*100 + '%' }}` using `bg-${palette color}`, rounded
      - Optional start/end value labels at bar edges
    - Compute `max` from the largest `end` value across all items
    - Color per row: `palette[i % palette.length]`
    - Card wrapper with --wf-* tokens (same as all chart components)
    - No ResponsiveContainer — pure div layout
  - **BumpChartComponent.tsx** — Multi-line ranking chart with reversed Y-axis
    - Props: `{ title: string, height?: number, categories?: string[], chartColors?: string[] }`
    - Mock data: 4 entities (Produto A-D) ranked across 6 periods (T1-T6) with rank crossings
      - Example: `[{ period: 'T1', 'Produto A': 1, 'Produto B': 3, 'Produto C': 2, 'Produto D': 4 }, ...]`
    - Implementation: `LineChart` with:
      - `<YAxis reversed domain={[1, 4]} tickCount={4} type="number">` — rank 1 at top
      - `<XAxis dataKey="period">` — time periods
      - One `<Line>` per entity with `type="monotone"`, `stroke={palette[i]}`, `strokeWidth={2.5}`, `isAnimationActive={false}`
      - Custom `dot` render prop: only render label on the LAST data point of each line — shows entity name to the right of the final dot
      - `<Tooltip>` for cross-entity comparison at each period
    - Custom legend with entity names and palette colors
    - Right margin padding on the chart to accommodate end-of-line labels (~60px)
