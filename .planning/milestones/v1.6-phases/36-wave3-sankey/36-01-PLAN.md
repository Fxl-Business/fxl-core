<objective>
Add the Sankey diagram as the 28th standalone section type in the wireframe builder, completing the 12-chart expansion of milestone v1.6. The Recharts Sankey named export is verified before any component code is written, and the implementation follows the 5-file checklist for standalone section types (XCUT-06).
</objective>

<context>
- Recharts 2.15.4 exports `Sankey` as a named function (verified: `Boolean(r.Sankey) === true`)
- Sankey expects `data: { nodes: Array<{name: string}>, links: Array<{source: number, target: number, value: number}> }`
- Links use **integer array indices** into the nodes array (not string names) — this is a hard API constraint
- Sankey does not use ResponsiveContainer well; it needs explicit width/height — use a measuring wrapper div
- The standalone section pattern requires: type definition, Zod schema, component + renderer, property form, and registry entry
- After Phase 35 the registry will have 27 entries; this phase brings it to 28
- chartColors from `useWireframeChartPalette` will color nodes; links inherit a transparent version of source node color
- Sankey has no `isAnimationActive` prop — it does not animate by default, satisfying XCUT-02 implicitly
</context>
