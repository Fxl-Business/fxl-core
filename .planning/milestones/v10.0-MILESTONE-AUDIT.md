## Integration Gaps (Must Fix)

### HIGH — Env var name inconsistency

The real codebase uses `VITE_SUPABASE_PUBLISHABLE_KEY` (see `src/platform/supabase.ts:4`). Several SDK pages incorrectly use `VITE_SUPABASE_ANON_KEY`:
- `ci-cd.md` (Vercel section, line ~237)
- `documentation.md` (README example, line ~70)
- `security.md` (env vars section, line ~241)
- `stack.md` (line ~127) — pre-existing, not from v10.0

**Fix:** Standardize all to `VITE_SUPABASE_PUBLISHABLE_KEY`.

### HIGH — Error boundary pattern contradiction

- `infrastructure.md` shows `Sentry.ErrorBoundary` functional wrapper
- `error-boundaries.md` mandates custom class component with `getDerivedStateFromError`

**Fix:** Update `infrastructure.md` to reference `/sdk/error-boundaries` for canonical pattern, or remove inline example.

### MEDIUM — SDK index stale status

`index.md` "Em Construcao" table still lists 8 v10.0 pages as incomplete.

**Fix:** Move completed pages to their respective sections with status `Completo`.

### MEDIUM — sort_order collisions in Supabase

- sort_order 80: code-standards, error-boundaries, mcp-server (3-way collision)
- sort_order 90: database, nexo-skill (2-way collision)

**Fix:** Assign unique sort_order values to each page.
