  <action>
    Create `tools/wireframe-builder/components/editor/ShareModal.tsx` -- a Dialog-based modal for managing share links.

    Props:
    ```typescript
    type Props = {
      open: boolean
      onClose: () => void
      clientSlug: string
      userId: string
    }
    ```

    Behavior per CONTEXT.md locked decisions:
    - On open (useEffect when open=true): call getTokensForClient(clientSlug) to load active tokens
    - Display list of active tokens, each showing: truncated token value, expiration date formatted as DD/MM/YYYY, a Revoke button
    - "Gerar novo link" button: calls createShareToken(clientSlug, userId, 30), appends result to local token list, shows toast.success('Link gerado!')
    - Each token row has a Copy button: constructs URL as `${window.location.origin}/wireframe-view?token=${token.token}&client=${clientSlug}`, copies via navigator.clipboard.writeText(), shows toast.success('Link copiado!')
    - Each token row has a Revoke button: calls revokeToken(token.id), removes from local list, shows toast.success('Link revogado!')
    - Loading state while fetching tokens
    - Empty state: "Nenhum link ativo" when no tokens

    UI components to use:
    - Dialog, DialogContent, DialogHeader, DialogTitle from shadcn
    - Button from shadcn for actions
    - Badge for token status/expiration
    - Copy, Trash2, Plus, Link2 icons from lucide-react
    - Use --wf-* inline styles for wireframe chrome consistency (this modal renders inside wireframe viewer, not inside Layout)

    Error handling: try/catch around each token operation, toast.error on failure with error message.

    IMPORTANT: Use existing tokens.ts functions directly. Do NOT create new token CRUD functions.
  </action>

    <automated>npx tsc --noEmit</automated>
  </verify>
