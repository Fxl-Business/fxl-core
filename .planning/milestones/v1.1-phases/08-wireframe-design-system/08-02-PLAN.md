<!-- Token mapping reference for migration. These are the Tailwind utilities from Plan 01 -->

Tailwind class mapping (hardcoded -> token):
- bg-white -> bg-wf-card
- bg-gray-50 -> bg-wf-canvas (or bg-wf-card depending on context)
- bg-gray-100 -> bg-wf-canvas
- bg-gray-200 -> (use for borders: border-wf-card-border)
- bg-gray-800, bg-gray-900 -> bg-wf-sidebar (sidebar), bg-wf-table-header (table headers)
- text-gray-400 -> text-wf-muted
- text-gray-500 -> text-wf-muted (or text-wf-body for readable text)
- text-gray-600 -> text-wf-body
- text-gray-700 -> text-wf-heading
- text-gray-800 -> text-wf-heading
- text-white (in dark bg contexts) -> text-wf-table-header-fg (tables), text-wf-sidebar-fg (sidebar)
- border-gray-200 -> border-wf-card-border
- border-gray-100 -> border-wf-card-border
- shadow-sm -> REMOVE entirely (user decision: no box-shadows)
- bg-green-50 text-green-700 -> bg-wf-positive/10 text-wf-positive (use opacity variant or inline style)
- bg-red-50 text-red-700 -> bg-wf-negative/10 text-wf-negative

Inline style hex mapping:
- #FFFFFF, #FFF -> var(--wf-card) or var(--wf-header-bg)
- #F5F5F5 -> var(--wf-canvas)
- #E0E0E0 -> var(--wf-card-border)
- #212121 -> var(--wf-heading)
- #424242 -> var(--wf-body)
- #757575 -> var(--wf-muted)
- #9E9E9E -> var(--wf-muted)
- #BDBDBD -> var(--wf-muted)
- #D1D5DB -> var(--wf-card-border)

Chart fill/stroke:
- Any hardcoded chart hex -> var(--wf-chart-1) through var(--wf-chart-5)
- chartColors prop references -> var(--wf-chart-N) where inside [data-wf-theme] container

Semantic colors:
- Green variants (positive indicators) -> var(--wf-positive)
- Red variants (negative indicators) -> var(--wf-negative)
</interfaces>
