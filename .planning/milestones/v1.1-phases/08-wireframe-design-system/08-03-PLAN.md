    3. Compute branding overrides at module level (next to existing brandVars):
       `const wfBrandOverrides = brandingToWfOverrides(branding)`
    4. Wrap the main wireframe container with WireframeThemeProvider. The current root div has `style={{ display: 'flex', height: '100vh', fontFamily: ..., background: '#F5F5F5', ...brandVars }}`. Restructure to:
       ```tsx
       <WireframeThemeProvider>
         <div style={{ display: 'flex', height: '100vh', fontFamily: `${branding.bodyFont}, Inter, sans-serif`, ...wfBrandOverrides }}>
           {/* sidebar, main content */}
         </div>
       </WireframeThemeProvider>
       ```
       The WireframeThemeProvider adds the data-wf-theme attribute via its own wrapper div. The inner div applies branding overrides which cascade into --wf-* tokens.
    5. Replace the sidebar's hardcoded inline colors:
       - `background: brandPalette.primaryDark` -> `background: 'var(--wf-sidebar-bg)'`
       - `color: '#FFF'` -> `color: 'var(--wf-sidebar-fg)'`
       - `borderBottom: 1px solid ${branding.primaryColor}` -> `borderBottom: '1px solid var(--wf-sidebar-border)'`
       - `borderTop: 1px solid ${branding.primaryColor}` -> `borderTop: '1px solid var(--wf-sidebar-border)'`
       - Active screen button: `background: resolvedBranding.primaryColor` -> `background: 'var(--wf-sidebar-active)'`, `color: '#FFF'` -> `color: 'var(--wf-sidebar-fg)'`
       - Inactive screen button: `color: '#BDBDBD'` -> `color: 'var(--wf-sidebar-muted)'`
       - Footer: `color: '#9E9E9E'/#757575` -> `color: 'var(--wf-sidebar-muted)'`
    6. Replace main area background: `background: '#F5F5F5'` -> `background: 'var(--wf-canvas)'` (on the root flex container)
    7. Remove the now-unnecessary `brandVars` spread from the root container. The branding is now applied through `wfBrandOverrides` which overrides --wf-accent and --wf-sidebar-* tokens. The old `--brand-*` vars from `brandingToCssVars` are no longer needed for wireframe rendering (keep the import for now in case other code uses it, but do not spread into the wireframe container).
    8. The chartColors prop passed to BlueprintRenderer can be kept for backward compat, but charts now also respond to var(--wf-chart-N) which is overridden by --wf-accent via branding. (Chart series 1 = --wf-accent which = branding.primaryColor when overrides applied.)
    9. CRITICAL: Ensure modals (pendingExitEdit, conflictOpen), CommentOverlay, CommentManager, and PropertyPanel render OUTSIDE the WireframeThemeProvider wrapper. These are app-level overlays and must keep app tokens. Move them after the closing WireframeThemeProvider tag. Use a Fragment or position them as siblings:
       ```tsx
       <>
         <WireframeThemeProvider>
           <div style={...}>
             {/* sidebar + main content */}
           </div>
         </WireframeThemeProvider>

         {/* App-level overlays - outside wireframe theme */}
         <PropertyPanel ... />
         {pendingExitEdit && <div className="fixed ...">...</div>}
         {conflictOpen && <div className="fixed ...">...</div>}
         {drawerTarget && <CommentOverlay ... />}
         <CommentManager ... />
         <button onClick={handleOpenScreenComments} className="fixed ...">...</button>
       </>
       ```
