# Phase 64 Plan 01: Multi-tenant Supabase Schema Migration

## Objective

Create a single SQL migration that makes all Supabase tables multi-tenant-ready with org_id columns, RLS policies filtering by org_id from JWT claims, and a new tenant_modules table.

## Context

- Design spec: docs/superpowers/specs/2026-03-16-fxl-platform-evolution-design.md (Section 5.4)
- Existing migrations: 001-007 in supabase/migrations/
- 7 existing tables: comments, share_tokens, blueprint_configs, briefing_configs, knowledge_entries, tasks, documents
- All currently use anon-permissive RLS policies

## Tasks

### Task 1: Create migration file `type="auto"`

Create `supabase/migrations/008_multi_tenant_schema.sql` with:

1. **tenant_modules table** - org_id text, module_id text, enabled boolean, config jsonb, PK (org_id, module_id)
2. **Add org_id** to all 7 existing tables - `text NOT NULL DEFAULT 'org_fxl_default'`
3. **Backfill** existing rows with 'org_fxl_default'
4. **Drop old RLS policies** - remove all anon-permissive policies from migrations 002-007
5. **Create new RLS policies** - org_id filtering with anon fallback using COALESCE pattern
6. **Create indexes** on org_id for all 8 tables

## Verification

- [ ] SQL is valid (CREATE TABLE, ALTER TABLE, DROP POLICY, CREATE POLICY, CREATE INDEX)
