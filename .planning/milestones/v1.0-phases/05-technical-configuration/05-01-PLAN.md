  <name>Task 2: Create config validator and pilot client TechnicalConfig</name>
  <files>tools/wireframe-builder/lib/config-validator.ts, clients/financeiro-conta-azul/wireframe/technical.config.ts</files>
  <action>
Create `tools/wireframe-builder/lib/config-validator.ts` implementing the validateConfig function per research Pattern 4.

**Exported types:**
- `ValidationError` with type ('missing-binding' | 'invalid-reference' | 'slug-mismatch' | 'orphan-binding' | 'circular-formula'), screenId, sectionIndex?, sectionType?, message
- `ValidationWarning` with type ('info-block-unbound'), screenId, sectionIndex?, message
- `ValidationResult` with valid: boolean, errors: ValidationError[], warnings: ValidationWarning[], coverage: { total: number, bound: number, percentage: number }

**Validation checks in order:**
1. **Slug match**: blueprint.slug must equal technical.slug. Error type: 'slug-mismatch'.
2. **Section coverage**: For each screen in blueprint, iterate sections. For each section, find a matching binding in technical.bindings by screenId + sectionIndex. If missing AND section type is NOT 'info-block', create error type 'missing-binding'. If section type is 'info-block' and no binding, create warning (info blocks are static content).
3. **ChartGrid recursion**: When section type is 'chart-grid', recurse into section.items to check each nested chart has a binding in the corresponding ChartGridBinding.items array.
4. **Field/formula reference validation**: For each binding, check that referenced fieldOrFormula IDs exist in technical.fields or technical.formulas. For each formula, check that references[] all point to existing field/formula IDs. Error type: 'invalid-reference'.
5. **Circular formula detection**: Build a dependency graph from formula references and perform topological sort. If cycle detected, error type: 'circular-formula'.
6. **Orphan binding detection**: For each binding in technical.bindings, check that the screenId exists in blueprint.screens and sectionIndex is within bounds. Error type: 'orphan-binding'.
7. **Coverage calculation**: total = count of all non-info-block sections (including chart-grid children), bound = count of those with valid bindings, percentage = Math.round(bound/total * 100).

Import types using `@tools/wireframe-builder/types/` path alias for blueprint and technical types (this is a lib file, not a type file).
