ComponentGallery.tsx is 649 lines with 39+ hardcoded gray-* references. This is a systematic token replacement exercise.

**Migration strategy** (per RESEARCH.md Pitfall 3):

1. **Toolbar/header areas:**
   - `bg-gray-100` / `bg-gray-50` -> `bg-muted`
   - `text-gray-800` / `text-gray-600` -> `text-foreground`
   - `text-gray-500` / `text-gray-400` -> `text-muted-foreground`
   - `border-gray-200` / `border-gray-300` -> `border-border`

2. **Component cards/previews:**
   - `bg-white` -> `bg-card`
   - `border-gray-200` -> `border-border`
   - `text-gray-900` -> `text-foreground`
   - `text-gray-500` -> `text-muted-foreground`
   - `shadow-sm` -> keep (shadows work in both modes)

3. **Interactive elements:**
   - Active/selected state colors -> use `bg-primary/10 text-primary` or `bg-accent/10`
   - Hover states -> `hover:bg-muted` or `hover:bg-muted/80`

4. **Status badges -- DO NOT MIGRATE these:**
   - `bg-green-50 text-green-700` (active/ready)
   - `bg-amber-50 text-amber-700` (warning/draft)
   - `bg-red-50 text-red-700` (error/deprecated)
   These are semantic status indicators, not theme colors.

5. Ensure the component gallery preview area (where components are rendered) uses `bg-card` or `bg-background` so component previews are visible in both modes.
