1. **Rewrite `CommentOverlay.tsx`** -- replace local state with Supabase. New props interface:
   ```typescript
   type Props = {
     clientSlug: string
     screenId: string
     targetId: string           // e.g. "screen:resultado-mensal-dfc" or "section:resultado-mensal-dfc:2"
     targetLabel?: string       // Human-readable label like "Tela: Resultado Mensal" or "Secao: KPI Grid"
     authorName: string         // From auth profile or localStorage
     authorRole: 'operador' | 'cliente'
     open: boolean
     onClose: () => void
   }
   ```
   Implementation:
   - On mount (and when `targetId` changes): fetch comments using `getCommentsByScreen(clientSlug, screenId)` and filter to those matching `targetId`. Store in local state.
   - Render the existing drawer UI (fixed right-0, w-80, border-l) but now:
     - Header shows `targetLabel` if provided, otherwise "Comentarios"
     - Each comment card shows `author_name` with a small label tag next to it: `<span class="...">Operador</span>` or `<span class="...">Cliente</span>` based on `author_role`. Use blue-100/blue-800 for Operador, amber-100/amber-800 for Cliente.
     - Resolved comments show "Resolvido" text in green-600 and get `opacity-50` styling (per user decision).
     - No resolve button here -- that's in CommentManager (Plan 03).
   - On submit: call `addComment({ clientSlug, targetId, authorName, authorRole, text: draft })`, then refetch comments to update the list.
   - The drawer is now a controlled component (`open` + `onClose` props instead of internal toggle state).
   - Remove the fixed FAB button (the parent will control when the drawer opens).

2. **Create `CommentIcon.tsx`** -- small icon that appears on hover for section-level comments:
   ```typescript
   type Props = { onClick: () => void }
   ```
   - Renders a `MessageSquare` icon (from lucide-react) in a small circle button
   - Positioned absolute, top-2 right-2 (parent must be relative)
   - Styling: bg-white shadow-sm border border-gray-200 rounded-full p-1.5, hover:bg-gray-50
   - Icon size: h-3.5 w-3.5 text-gray-500
   - Transition: opacity-0 group-hover:opacity-100 (appears only on parent hover)

3. **Create `CommentBadge.tsx`** -- small badge showing unresolved comment count:
   ```typescript
   type Props = { count: number }
   ```
   - If count is 0, render nothing (return null)
   - Otherwise render a small circle with the count, positioned absolute top-1 right-1
   - Styling: bg-amber-500 text-white text-[9px] font-bold rounded-full min-w-[18px] h-[18px] flex items-center justify-center
   - Subtle, not disruptive (per user decision: "small, corner-positioned")
