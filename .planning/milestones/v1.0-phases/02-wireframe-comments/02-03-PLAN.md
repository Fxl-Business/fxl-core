1. **Create `src/pages/SharedWireframeView.tsx`** -- the token-gated wireframe viewer for external clients.

   The page has 4 states, shown sequentially:
   - **Loading**: validating token (show centered spinner + "Validando acesso...")
   - **Invalid**: token missing, expired, or revoked (show error card with message "Link invalido ou expirado. Solicite um novo link ao operador.")
   - **Name entry**: token valid, no name in localStorage (show centered card with text input "Seu nome" + "Comecar" button)
   - **Wireframe**: authenticated and named, show the full wireframe viewer

   Implementation flow (following RESEARCH.md Pattern 4):
   a. Read `token` from URL search params (`useSearchParams`)
   b. If no token: show Invalid state immediately
   c. Call `validateToken(token)` -- if invalid: show Invalid state
   d. If valid: call `supabase.auth.signInAnonymously()` -- this gives the client a real `auth.uid()`
   e. Check localStorage for `fxl_client_name` -- if missing: show Name entry state
   f. On name submit: save to localStorage, transition to Wireframe state
   g. In Wireframe state: dynamically import the client's blueprint config based on `clientSlug` from the token

   For the Wireframe state, render the same layout as `FinanceiroWireframeViewer` (as updated by Plan 02):
   - Dark sidebar with screen list
   - WireframeHeader + BlueprintRenderer in main area
   - Screen-level comment FAB (same as WireframeViewer)
   - CommentOverlay drawer with `authorRole="cliente"` and `authorName` from localStorage
   - BlueprintRenderer with `clientSlug`, `comments`, and `onOpenComments` props (added by Plan 02)

   **Important**: For v1, since only one client exists (financeiro-conta-azul), hardcode the blueprint import. Use a mapping object:
   ```typescript
   const blueprintMap: Record<string, () => Promise<{ default: BlueprintConfig }>> = {
     'financeiro-conta-azul': () => import('@clients/financeiro-conta-azul/wireframe/blueprint.config'),
   }
   ```
   Load the blueprint dynamically based on `clientSlug` from the validated token. Show "Cliente nao encontrado" if slug not in map.

   **Client comment integration**: The client can open the CommentOverlay per screen (via FAB) and per section (via SectionWrapper created in Plan 02). Pass `authorRole="cliente"` and `authorName` from localStorage. No resolve capability for clients.
