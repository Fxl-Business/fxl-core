  <action>
Migrate 4 chart components + their section renderer to accept brand colors as props.

CRITICAL: Recharts SVG fill/stroke attributes do NOT support CSS var() syntax. Pass resolved hex color strings as props to chart components. Do NOT use var(--brand-*) in recharts fill or stroke attributes.

**BarLineChart.tsx:**
- Add optional prop `chartColors?: string[]` to Props type
- Replace hardcoded `fill="#d1d5db"` on Bar with `fill={chartColors?.[0] ?? '#d1d5db'}`
- Replace hardcoded `stroke="#9ca3af"` on Line with `stroke={chartColors?.[1] ?? '#9ca3af'}`
- Same for composed chart variant if it has different hardcoded colors
- Keep all other behavior identical

**DonutChart.tsx:**
- Add optional prop `chartColors?: string[]` to Props type
- Replace hardcoded `COLORS = ['#212121', '#757575', '#BDBDBD', '#424242', '#9E9E9E']`
- Use `chartColors ?? COLORS` as the palette array
- Keep existing indexing pattern `palette[index % palette.length]`

**WaterfallChart.tsx:**
- Add optional prop `chartColors?: { primary: string; accent: string }` to Props type
- Replace `FILL.subtotal = '#3b82f6'` with `chartColors?.primary ?? '#3b82f6'`
- Replace `FILL.positive = '#22c55e'` with `chartColors?.accent ?? '#22c55e'`
- KEEP `FILL.negative = '#ef4444'` unchanged -- semantic red for negative is NOT branded (per user decision: semantic status colors stay as-is)
- Apply same pattern to FILL_COMPARE variants (lighter versions)

**ParetoChart.tsx:**
- Add optional prop `chartColors?: { primary: string; accent: string }` to Props type
- Replace `fill="#374151"` on Bar with `fill={chartColors?.primary ?? '#374151'}`
- Replace `stroke="#DC2626"` on Line with `stroke={chartColors?.accent ?? '#DC2626'}`
- Replace `fill="#DC2626"` on Line dot with same accent color

**ChartRenderer.tsx:**
- Add optional prop `chartColors?: string[]` to Props type
- Pass `chartColors` to BarLineChart and DonutChart
- For WaterfallChart and ParetoChart, derive `{ primary: chartColors[0], accent: chartColors[2] }` from the array
- All props optional -- when undefined, components use their existing hardcoded fallbacks (zero visual change)
