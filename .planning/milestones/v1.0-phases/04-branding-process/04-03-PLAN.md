<task type="auto">
  <name>Task 2: Inject branding into WireframeViewer and SharedWireframeView</name>
  <files>src/pages/clients/FinanceiroContaAzul/WireframeViewer.tsx, src/pages/SharedWireframeView.tsx</files>
  <action>
Wire branding at the top-level viewer components:

**WireframeViewer.tsx:**
1. Import branding utilities: `import { resolveBranding, brandingToCssVars, getChartPalette, getFontLinks } from '@tools/wireframe-builder/lib/branding'`
2. Import pilot branding config: `import brandingConfigSeed from '@clients/financeiro-conta-azul/wireframe/branding.config'`
3. Resolve branding at component level: `const branding = resolveBranding(brandingConfigSeed)`
4. Generate CSS vars: `const brandVars = brandingToCssVars(branding)`
5. Generate chart palette: `const chartPalette = getChartPalette(branding)`
6. Inject on root container div:
   - Spread `...brandVars` into the existing style object
   - Set `fontFamily` to `${branding.bodyFont}, Inter, sans-serif`
7. Update sidebar:
   - Sidebar background: derive a very dark variant from primaryColor. Use `--brand-primary-dark` or compute inline: if primary is light (HSL L > 25), darken to L=15; if already dark, use directly. Import `darken` from branding lib if needed, or use the CSS var `var(--brand-primary-dark, #212121)` since sidebar is not SVG.
   - Sidebar border: use `var(--brand-primary, #424242)` with reduced opacity or a slightly lighter shade
   - Logo section: check `branding.logoUrl` -- if truthy, render `<img src={branding.logoUrl} alt={config.label} style={{ maxHeight: 32, maxWidth: 120, objectFit: 'contain' }} />`; if falsy, keep current `<span>FXL</span>` text
   - Active screen button bg: use `var(--brand-primary, #424242)` with some transparency
8. Pass brand colors to BlueprintRenderer: `chartColors={chartPalette}` and `brandPrimary={branding.primaryColor}`
9. Font loading: add useEffect that injects Google Font `<link>` tags using `getFontLinks(branding)`. Create link elements, append to document.head, clean up on unmount.
10. Favicon: if `branding.faviconUrl` is set, update `document.querySelector('link[rel="icon"]')?.setAttribute('href', url)` in useEffect

**SharedWireframeView.tsx:**
1. Same branding imports as WireframeViewer
2. For SharedWireframeView, the branding config needs to come from the client slug. Since we're keeping branding as .ts files (not Supabase), create a `brandingMap` similar to the existing `blueprintMap`:
   ```typescript
   const brandingMap: Record<string, () => Promise<{ default: BrandingConfig }>> = {
     'financeiro-conta-azul': () => import('@clients/financeiro-conta-azul/wireframe/branding.config'),
   }
   ```
3. Import BrandingConfig type from types/branding
4. Load branding in `loadBlueprint` function alongside blueprint: `const brandMod = await brandingMap[clientSlug]?.()` -- if undefined, use DEFAULT_BRANDING
5. Store branding in viewState (add branding field to 'wireframe' step state)
6. Apply same CSS var injection, sidebar styling, logo, font loading, and prop passing as WireframeViewer
7. All sidebar colors and logo follow same pattern as WireframeViewer

IMPORTANT:
- Only inject `--brand-*` CSS vars on the wireframe container div, NOT on document.body or #root
- The FXL Core app theme (`--primary`, `--accent`, etc. in globals.css) must NOT be affected
- Use `--brand-*` prefix exclusively for branding variables
