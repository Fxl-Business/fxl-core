<decisions>
## Implementation Decisions

### Taxonomia — 4 secoes top-level
- **Processo** — regras normativas, fases, identidade, fluxos (sem mudancas estruturais)
- **Padroes** — NOVA secao. Agrupa: Tech Radar + 15 techs + Premissas Gerais + Seguranca + Testes. Tudo que e "regra base que todo app deve seguir" vive aqui
- **Ferramentas** — so Wireframe Builder (com Blocos Disponiveis + Galeria de Componentes como sub-itens)
- **Clientes** — sem mudancas
- "Referencias" deixa de existir como secao. Seus itens sao redistribuidos

### Paginas removidas
- **Biblioteca de KPIs** — remover completamente (redundante com Blocos Disponiveis)
- **Master Prompt / Template de Sprint** — remover como pagina standalone. Substituir por documentacao de como implementar GSD nos projetos
- **CLAUDE.md Template** — remover como pagina standalone. Mover conteudo para dentro do processo de criacao de projeto FXL

### Reescrita de conteudo
- Reescrita completa de todos os docs de processo (6 fases + visao geral + prompts + onboarding)
- Mudanca principal: Claude Project → Claude Code + Skills + Agents + GSD
- As 6 fases (Diagnostico, Wireframe, Desenvolvimento, Auditoria, Entrega, Tutorial) permanecem iguais em escopo — so os docs descritivos estao desatualizados
- Tom: misto conforme contexto — processo/fases = operacional e direto, padroes/techs = referencia tecnica

### Estrutura de navegacao
- Todo parent item com children deve ter landing page propria
- Landing pages faltantes que devem ser criadas: **Fases** (resumo + cards das 6 fases), **Padroes** (index da nova secao)
- Regra: landing page = resumo do que sao os sub-itens + links para cada um
