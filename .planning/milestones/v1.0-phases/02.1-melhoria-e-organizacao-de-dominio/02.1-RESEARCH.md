## Architecture Patterns

### Current System Architecture (Verified from Source)

The documentation platform renders `.md` files from `docs/` using a custom parser pipeline:

```
docs/*.md  -->  docs-parser.ts (frontmatter + custom tags)  -->  DocRenderer.tsx  -->  React UI
                                                                      |
Sidebar.tsx (hardcoded navigation array)                              |
DocBreadcrumb.tsx (sectionSlugMap for badge -> index linking)         |
search-index.ts (builds index from frontmatter badges)               |
App.tsx (routes: /processo/*, /referencias/*, /ferramentas/*)        |
```

**Key patterns to follow:**
1. **Navigation is hardcoded** in `Sidebar.tsx` as a `NavItem[]` array. No dynamic generation. To change nav, edit the array directly.
2. **Badges determine visual grouping.** The `badge` field in frontmatter controls: (a) the header badge on each page, (b) breadcrumb section linking via `DocBreadcrumb.tsx`, (c) search index categorization.
3. **Routes are path-based.** DocRenderer uses `location.pathname` to resolve `/docs/{path}.md`. Files at `/docs/ferramentas/techs/supabase.md` serve at route `/ferramentas/techs/supabase`.
4. **Landing pages use `{% phase-card %}` tags** for card-based navigation to sub-pages.
5. **Special routes override DocRenderer.** The ComponentGallery at `/ferramentas/wireframe-builder/galeria` has its own route in `App.tsx` that takes precedence over the wildcard.

### Critical Routing Issue: Padroes Landing Page

**Problem:** The new "Padroes" section needs a landing page, but there is no `docs/padroes/` directory. Files like `premissas-gerais.md` live in `docs/ferramentas/` and tech pages live in `docs/ferramentas/techs/`. The constraint says "no file moves."

**Solution options:**
1. **Create `docs/padroes/index.md` as a standalone landing page** that links to files in their current locations (`/ferramentas/premissas-gerais`, `/ferramentas/tech-radar`, etc.). The sidebar hrefs already point to the actual file paths. The landing page is just an index with phase-cards.
2. Add a route `/padroes/index` in App.tsx that serves `docs/padroes/index.md`.

**Recommendation:** Option 1. Create a minimal `docs/padroes/` directory with just `index.md`. This is not "moving files" -- it is creating one new file. The route `/padroes/*` handles it. All other files stay where they are; sidebar entries point to their actual paths.

## Common Pitfalls

### Pitfall 1: Route Mismatch After Badge Changes
**What goes wrong:** Changing a file's badge to "Padroes" but forgetting to add "Padroes" to `sectionSlugMap` in DocBreadcrumb.tsx. Result: breadcrumb shows plain text instead of clickable link.
**How to avoid:** Update `sectionSlugMap` in the same commit as badge changes. Add entry: `Padroes: '/padroes/index'`.

### Pitfall 2: Sidebar href vs Actual File Path Divergence
**What goes wrong:** Sidebar entries like `{ label: 'Premissas Gerais', href: '/padroes/premissas-gerais' }` but the file is actually at `docs/ferramentas/premissas-gerais.md` (serving at `/ferramentas/premissas-gerais`). DocRenderer would return 404.
**How to avoid:** Since files are NOT being moved, sidebar hrefs MUST point to the actual serving path. Example: `{ label: 'Premissas Gerais', href: '/ferramentas/premissas-gerais' }`. The section grouping in sidebar is independent of the file path.

### Pitfall 3: Broken Internal Links in Content
**What goes wrong:** Content rewrites in fase docs reference pages by old paths (e.g., `/referencias/biblioteca-kpis`) that have been removed.
**How to avoid:** After removing pages, grep all .md files for references to removed paths. Replace with new locations or remove the references.

### Pitfall 4: Orphaned Routes
**What goes wrong:** After removing `docs/referencias/index.md`, the route `/referencias/*` in App.tsx still exists, potentially serving 404 pages for the removed index.
**How to avoid:** The `/referencias/*` route should be kept because `docs/referencias/blocos-disponiveis.md` still exists at that path. Only remove the route if ALL files under that directory are moved or deleted.

### Pitfall 5: Content Rewrite Losing Prompts
**What goes wrong:** Rewriting fase docs and accidentally removing `{% prompt %}` blocks that are actively used by operators.
**How to avoid:** Before rewriting, catalog all existing `{% prompt %}` blocks. After rewriting, verify each prompt still exists and is contextually correct for the new Claude Code + GSD workflow.

### Pitfall 6: Inconsistent References Across Docs
**What goes wrong:** Onboarding page references "Template de Sprint" but that page has been removed. Fase 1 references "Biblioteca de KPIs" but that page has been removed.
**How to avoid:** After all removals and rewrites, run a link audit across all .md files checking for references to removed pages.
