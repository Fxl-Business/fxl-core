**1. EditableSectionWrapper.tsx**

This is the core edit mode wrapper around each section. Uses @dnd-kit/sortable for drag-and-drop.

Props:
```typescript
type Props = {
  id: string                    // unique ID for dnd-kit (row.id or section-specific)
  editMode: boolean
  selected: boolean             // currently selected for property editing
  onSelect: () => void          // click to open property panel
  onDelete: () => void          // remove this section
  children: React.ReactNode
}
```

Implementation:
- Use `useSortable({ id })` from @dnd-kit/sortable
- Apply `CSS.Transform.toString(transform)` and `transition` to the outer div
- When `editMode` is true:
  - Outer div gets `border-2 border-dashed border-muted-foreground/30 rounded-lg relative group` and `cursor-pointer`
  - When `selected`: `border-primary ring-2 ring-primary/20`
  - On click (the outer div, not the drag handle): call `onSelect`
  - Drag handle: absolute positioned on the left, GripVertical icon, `opacity-0 group-hover:opacity-100` transition. Apply `{...attributes} {...listeners}` from useSortable to the drag handle button only (NOT the outer div).
  - Delete button: absolute positioned top-right, Trash2 icon, `opacity-0 group-hover:opacity-100`, red on hover. On click: show a small inline confirmation ("Remover?" with Confirmar/Cancelar) using local state -- NOT a full dialog. Use `e.stopPropagation()` to prevent selecting the section.
  - When `isDragging`: `opacity-50`
- When `editMode` is false: render children directly without any wrapper controls

Icons: GripVertical, Trash2 from lucide-react.

Define a constant `SECTION_CATALOG` inside this file mapping each type to its category, label, and icon component.
