**2. All 15 property forms** (in `property-forms/` subdirectory):

Each form follows this contract:
```typescript
type Props = {
  section: SpecificSectionType    // e.g., KpiGridSection
  onChange: (updated: SpecificSectionType) => void
}
```

Each form uses shadcn Input, Label, Select where appropriate. All forms call `onChange({...section, field: newValue})` on every field change (controlled components, live preview).

**Form details for each type:**

- **KpiGridForm**: Input for columns (number), groupLabel (text). List of KPI items: each item has label (Input), value (Input), sub (Input), variation (Input). Add/remove item buttons.
- **BarLineChartForm**: Input for title (text). Select for chartType (bar/line/bar-line). Input for height (number, optional).
- **DonutChartForm**: Input for title.
- **WaterfallChartForm**: Input for title. Input for height (optional). List of bars: each has label (Input), value (Input, number). Add/remove bar buttons.
- **ParetoChartForm**: Input for title.
- **CalculoCardForm**: Input for title. Note "Rows are demo data" (no row editing in v1 -- rows are complex nested types).
- **DataTableForm**: Input for title. Input for rowCount (number). List of columns: each has key (Input), label (Input), align (Select: left/right/center). Add/remove column buttons.
- **DrillDownTableForm**: Input for title. Input for subtitle. List of columns: key (Input), label (Input). Note "Rows are demo data" for v1.
- **ClickableTableForm**: Input for title. Input for subtitle. List of columns: key (Input), label (Input). Note "Rows are demo data" for v1.
- **SaldoBancoForm**: Input for title (optional). Input for note (optional). Input for total. List of banks: label (Input), value (Input). Add/remove.
- **ManualInputForm**: Input for title. Input for initialBalance (optional).
- **UploadSectionForm**: Input for label. Input for successMessage (optional). Note about acceptedFormats.
- **ConfigTableForm**: Input for title. Input for addLabel (optional). List of columns with add/remove.
- **InfoBlockForm**: Textarea for content. Select for variant (info/warning).
- **ChartGridForm**: Input for columns (number). Note "Sub-items are managed via the main editor" (chart-grid contains nested BlueprintSection[] -- too complex for inline editing in v1).

**Pattern for list editing** (used in KpiGridForm, DataTableForm, SaldoBancoForm, etc.):
```typescript
function handleAddItem() {
  onChange({ ...section, items: [...section.items, defaultItem] })
}
function handleRemoveItem(index: number) {
  onChange({ ...section, items: section.items.filter((_, i) => i !== index) })
}
function handleUpdateItem(index: number, field: string, value: string) {
  onChange({
    ...section,
    items: section.items.map((item, i) => i === index ? { ...item, [field]: value } : item)
  })
}
```

Use Tailwind `space-y-4` for form field spacing. Each field: Label above, Input/Select below. Lists have a subtle border with `p-3 rounded-md border space-y-2` per item and a "+" button at the bottom.

For complex types that have array data intended as demo (CalculoCard rows, DrillDownTable rows, ClickableTable rows, ChartGrid items): show a read-only note "Dados de exemplo -- editar no config" rather than building complex nested editors. Focus editing on the structural properties (title, columns, layout).

**1. ScreenManager.tsx**

Props:
```typescript
type Props = {
  screens: BlueprintScreen[]
  activeIndex: number
  editMode: boolean
  onSelectScreen: (index: number) => void
  onAddScreen: (screen: BlueprintScreen) => void
  onDeleteScreen: (index: number) => void
  onRenameScreen: (index: number, title: string) => void
  onReorderScreens: (screens: BlueprintScreen[]) => void
}
```

This replaces the inline sidebar nav that currently exists in WireframeViewer. Renders the screen list in the dark sidebar area.

When `editMode` is false: render the screen list exactly like the current sidebar (button per screen, active state highlight). This is the baseline behavior.

When `editMode` is true, add:
- Drag handles (GripVertical) on each screen item for reordering. Use @dnd-kit/sortable with DndContext + SortableContext + verticalListSortingStrategy. On DragEnd, call onReorderScreens with the reordered array (use arrayMove from @dnd-kit/sortable).
- "..." menu (MoreVertical icon) on each screen item. On click, show a small dropdown/popover with two options:
  - "Renomear": Switches the screen title to an inline input. On Enter or blur, call onRenameScreen.
  - "Excluir": Shows a small confirmation inline: "Esta tela tem N secoes. Deseja excluir?" with Confirmar/Cancelar buttons. On confirm, call onDeleteScreen.
- "+" button at the bottom of the screen list to add a new screen. On click, opens a Dialog with:
  - Input for title (required)
  - IconPicker for icon (optional, defaults to 'layout-dashboard')
  - Select for periodType ('mensal', 'anual', 'none')
  - "Criar Tela" button that calls onAddScreen with a new BlueprintScreen: `{ id: crypto.randomUUID(), title, icon, periodType, filters: [], hasCompareSwitch: false, sections: [], rows: [] }`

Styling: dark sidebar theme matching existing wireframe sidebar (bg-[#212121], text-white). Active screen: bg-[#424242]. Edit controls: subtle, not overwhelming.
