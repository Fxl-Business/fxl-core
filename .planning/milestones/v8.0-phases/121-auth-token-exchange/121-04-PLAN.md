  <action>
Update `supabase/functions/auth-token-exchange/index.ts` to extract and forward the
`super_admin` claim from the Clerk JWT to the minted Supabase JWT.

Clerk stores `super_admin` in `publicMetadata`, which appears in the session token payload.
The claim path in the Clerk JWT depends on the JWT template — it may be:
- `payload.super_admin` (if a custom JWT template was set up to include it at the top level)
- `(payload as any).public_metadata?.super_admin` (if using default session token)

Add this extraction after the existing claim extractions (around line 68-70):
```typescript
// Extract super_admin from Clerk JWT (stored in publicMetadata)
// Try top-level first (custom JWT template), then publicMetadata path
const superAdmin = (
  (payload as Record<string, unknown>).super_admin === true ||
  ((payload as Record<string, unknown>).public_metadata as Record<string, unknown> | undefined)?.super_admin === true
)
```

Add to the SignJWT payload:
```typescript
const supabaseJwt = await new jose.SignJWT({
  sub: userId,
  org_id: orgId,
  role: orgRole ?? 'authenticated',
  aud: 'authenticated',
  iss: 'supabase',
  ...(superAdmin ? { super_admin: true } : {}),
})
```

The file has `@ts-nocheck` at the top — TypeScript errors in the file itself are suppressed.
However, confirm the logic is syntactically correct by reviewing it.
