1. **Race condition on org switch**: When the user switches orgs, `isReady` stays `true` briefly
   while the new token exchange is in flight. During that window, components can fire Supabase
   queries with the old org's token — RLS sees the wrong org_id.

   Tests importing `@platform/*`, `@shared/*`, `@modules/*` fail to resolve. All unit tests for
   platform code (TEST-01) are blocked until this is fixed.

Purpose: Unblock test authoring (Plan 121-03) and eliminate the most likely cause of stale data
appearing after org switch.

Output: Updated `vitest.config.ts` and `useOrgTokenExchange.ts` with race condition fix.
</objective>

<task type="auto">
  <name>Task 2: Fix race condition in useOrgTokenExchange — reset isReady on org change</name>
  <files>src/platform/tenants/useOrgTokenExchange.ts</files>
  <action>
Fix the race condition where `isReady` stays `true` during the in-flight token exchange
for a new org, allowing queries to fire with the previous org's token.

**The bug:** When `activeOrg?.id` changes, the effect re-runs `doExchange()`. But between
when the effect starts and when `setIsReady(true)` is called, `isReady` is still `true`
from the previous org. ProtectedRoute shows children, components fire queries, RLS sees
the wrong org_id.

**The fix:** At the very start of `doExchange()`, call `setIsReady(false)` and
`setOrgAccessToken(null)` so the gate closes immediately when a new exchange begins.
Also call these synchronously BEFORE the async operations start.

Current code:
```typescript
async function doExchange() {
  try {
    const clerkToken = await session!.getToken()
    // ...
    setOrgAccessToken(result.access_token)
    setError(null)
    setIsReady(true)
    // ...
  } catch (err) {
    // ...
    setIsReady(false)
    setOrgAccessToken(null)
  }
}
```

Change to:
```typescript
async function doExchange() {
  // Immediately invalidate current token so no queries fire with stale org token
  setIsReady(false)
  setOrgAccessToken(null)
  setError(null)

  try {
    const clerkToken = await session!.getToken()
    if (!clerkToken) {
      setError('Clerk session token unavailable')
      return
    }

    const result = await exchangeToken(clerkToken, activeOrg!.id)
    setOrgAccessToken(result.access_token)
    setIsReady(true)

    if (orgChanged && onOrgChangeRef.current) {
      onOrgChangeRef.current()
    }

    prevOrgIdRef.current = activeOrg!.id
  } catch (err) {
    const message = err instanceof Error ? err.message : 'Token exchange failed'
    setError(message)
    console.error('[useOrgTokenExchange]', message)
  }
}
```

Note: Remove the duplicate `setError(null)` and `setOrgAccessToken(null)` from the catch
block since they are now set at the top. Keep `setError(message)` in catch.

Also fix the initial state: the line
```typescript
const [isReady, setIsReady] = useState(() => getOrgAccessToken() !== null)
```
should remain as-is — it's correct for SPA navigation where token is already in memory.

Run `npx tsc --noEmit` after changes to confirm zero errors.
