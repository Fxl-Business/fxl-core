Add 3 new mutation helpers to `WireframeViewerInner` in `WireframeViewer.tsx`, placed immediately after the existing `updateWorkingScreen` function (after line 470). These helpers follow the EXACT same pattern as `updateWorkingScreen`:

**1. `updateWorkingConfig`** — generic dashboard-level config mutator:

```typescript
function updateWorkingConfig(
  updater: (config: BlueprintConfig) => BlueprintConfig,
) {
  setWorkingConfig((prev) => {
    if (!prev) return prev
    return updater(prev)
  })
  setEditMode((prev) => ({ ...prev, dirty: true }))
}
```

This is the dashboard-level counterpart to `updateWorkingScreen`. It:
- Takes a pure updater function (same functional update pattern)
- Uses `setWorkingConfig` with functional form (no stale closures)
- Returns `prev` unchanged when `prev` is null (edit mode not active guard)
- Always sets `dirty: true` on editMode (triggers save button)

**2. `updateWorkingSidebar`** — convenience for sidebar mutations:

```typescript
function updateWorkingSidebar(patch: Partial<SidebarConfig>) {
  updateWorkingConfig((cfg) => ({
    ...cfg,
    sidebar: { ...cfg.sidebar, ...patch },
  }))
}
```

**3. `updateWorkingHeader`** — convenience for header mutations:

```typescript
function updateWorkingHeader(patch: Partial<HeaderConfig>) {
  updateWorkingConfig((cfg) => ({
    ...cfg,
    header: { ...cfg.header, ...patch },
  }))
}
```
