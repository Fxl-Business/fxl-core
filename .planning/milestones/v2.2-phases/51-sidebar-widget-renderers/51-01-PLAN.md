Create the `SIDEBAR_WIDGET_REGISTRY` constant that maps each `SidebarWidgetType` to its rendering metadata. This registry is consumed by WireframeViewer to look up icons for collapsed rendering and by future components (Phase 52) for widget picker labels.

**Create `tools/wireframe-builder/lib/sidebar-widget-registry.ts`**

```typescript
import type { ComponentType } from 'react'
import type { SidebarWidgetType } from '../types/blueprint'
import { ChevronsUpDown, User, Building2, Search } from 'lucide-react'

export type SidebarWidgetZone = 'header' | 'nav' | 'footer'

export type SidebarWidgetRegistration = {
  icon: ComponentType<{ style?: React.CSSProperties; className?: string }>
  label: string
  zone: SidebarWidgetZone
}

export const SIDEBAR_WIDGET_REGISTRY: Record<SidebarWidgetType, SidebarWidgetRegistration> = {
  'workspace-switcher': {
    icon: ChevronsUpDown,
    label: 'Workspace Switcher',
    zone: 'header',
  },
  'user-menu': {
    icon: User,
    label: 'User Menu',
    zone: 'footer',
  },
  'account-selector': {
    icon: Building2,
    label: 'Account Selector',
    zone: 'header',
  },
  'search': {
    icon: Search,
    label: 'Search',
    zone: 'nav',
  },
}
```

Key design decisions:
- `zone` field determines WHERE in the sidebar the widget renders (header/nav/footer). This is derived from the widget type, not from `widget.position` — the zone is fixed per widget type. The `position: 'top' | 'bottom'` field on `SidebarWidget` is a secondary ordering hint within a zone; `zone` is the structural assignment.
- Icon components are imported directly from `lucide-react` — same pattern as `section-registry.tsx`
- `ChevronsUpDown` for workspace-switcher (matches shadcn sidebar-07 team-switcher pattern)
- `User` for user-menu (standard user/avatar icon)
- `Building2` and `Search` are registered for completeness but not rendered in Phase 51 (only workspace-switcher and user-menu are in scope per SIDE-04 and SIDE-05)
- The type uses `ComponentType<{ style?: React.CSSProperties; className?: string }>` which is compatible with Lucide icon components without importing `LucideIcon` (avoids coupling to lucide-react internals)

CRITICAL: Do NOT use `any`. The `Record<SidebarWidgetType, ...>` type ensures exhaustive coverage of all widget types.
