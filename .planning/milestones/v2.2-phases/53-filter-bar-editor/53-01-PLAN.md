```typescript
function updateWorkingScreen(
  updater: (screen: BlueprintScreen) => BlueprintScreen,
) {
  setWorkingConfig((prev) => {
    if (!prev) return prev
    const newScreens = [...prev.screens]
    const screen = newScreens[safeActiveIndex]
    if (!screen) return prev
    newScreens[safeActiveIndex] = updater(screen)
    return { ...prev, screens: newScreens }
  })
  setEditMode((prev) => ({ ...prev, dirty: true }))
}
```

**Component structure:**

The Sheet opens from the right side (matching PropertyPanel pattern). It contains:

1. **Header section** тАФ Sheet title "Filtros da Tela" with description "Adicione e configure os filtros da barra fixa desta tela."

2. **Preset buttons section** тАФ A horizontal row of 5 compact buttons labeled "Periodo", "Empresa", "Produto", "Status", "Responsavel". Each button has a `+` icon prefix. Clicking a preset button calls `onChange` with the current filters array plus the new preset FilterOption appended. The section has a label "Adicionar Predefinido" and a Separator below it.

3. **Filter list section** тАФ Each existing filter rendered as a card with:
   - **Header row:** Filter key displayed as a read-only Badge (e.g., `empresa`), plus a delete button (Trash2 icon) on the right
   - **Label field:** Input for `filter.label`, editable, updates on change
   - **FilterType field:** Select dropdown with 5 options:
     - `select` тАФ "Select"
     - `date-range` тАФ "Intervalo de Data"
     - `multi-select` тАФ "Multi-Select"
     - `search` тАФ "Busca"
     - `toggle` тАФ "Toggle"
   - **Options field:** Only shown when filterType is `select` or `multi-select`. An Input where options are entered as comma-separated values. Display current options as comma-joined string. On change, split by comma, trim, filter empty strings.
   - Each field change immediately calls `onChange` with the updated filters array (live preview)

4. **Add new filter section** тАФ At the bottom, a bordered card with:
   - Input for key (slug format, lowercase, no spaces тАФ display placeholder "chave-do-filtro")
   - Input for label (display placeholder "Label do filtro")
   - Select for filterType (same 5 options as above, default "select")
   - Input for options (comma-separated, only shown when filterType is `select` or `multi-select`, placeholder "Opcao 1, Opcao 2, ...")
   - Button "Adicionar Filtro" (Plus icon) тАФ disabled when key is empty or key already exists in current filters
   - On click: append new FilterOption to filters array, clear the form inputs, call onChange
