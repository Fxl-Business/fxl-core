<objective>
Extend the SidebarConfig schema with a SidebarWidget discriminated union type in both TypeScript and Zod. This is the foundational schema change that all subsequent v2.2 phases depend on. After this plan completes, any component can import `SidebarWidget` and `SidebarWidgetType` from `types/blueprint.ts`, and any `SidebarConfig` with a `widgets` array will survive Zod validation and DB round-trips. Existing blueprints without widgets continue to parse without errors.

Purpose: INFRA-03 is the hard dependency for all other v2.2 phases. No sidebar widget renderer (Phase 51), no sidebar config panel (Phase 52), and no widget picker can be built until these types exist.
Output: TypeScript types + Zod schemas + test coverage confirming backward compatibility and new field validation.
</objective>
