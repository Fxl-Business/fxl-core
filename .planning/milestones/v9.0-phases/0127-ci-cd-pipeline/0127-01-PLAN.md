## Objective

Create a GitHub Actions workflow that runs `tsc --noEmit` and `vitest run` on every PR targeting main, then configure branch protection to block merges when CI fails.

## Tasks

<task id="1">
<title>Create GitHub Actions CI workflow file</title>
<read_first>
- package.json (verify script names: "test" = "vitest run", "build" includes "tsc --noEmit")
- vitest.config.ts (confirm test include patterns and setup file path)
- tsconfig.json (confirm TypeScript config used by tsc --noEmit)
</read_first>
<action>
Create `.github/workflows/ci.yml` with the following content:

```yaml
name: CI

on:
  pull_request:
    branches: [main]
    types: [opened, synchronize, reopened]

concurrency:
  group: ci-${{ github.ref }}
  cancel-in-progress: true

jobs:
  type-check-and-test:
    name: Type Check & Test
    runs-on: ubuntu-latest

    steps:
      - name: Checkout
        uses: actions/checkout@v4

      - name: Setup Node.js
        uses: actions/setup-node@v4
        with:
          node-version: 20
          cache: npm

      - name: Install dependencies
        run: npm ci

      - name: Type check
        run: npx tsc --noEmit

      - name: Run tests
        run: npx vitest run
```

Key decisions embedded in this file:
- `pull_request` trigger on `main` with `opened`, `synchronize`, `reopened` event types
- `concurrency` group with `cancel-in-progress: true` to cancel stale runs on new pushes
- Single job `type-check-and-test` with sequential steps
- Node 20 matching local dev environment (v20.18.2)
- `actions/setup-node@v4` with `cache: npm` for dependency caching
- `npm ci` for clean reproducible installs
- `npx tsc --noEmit` for type checking (matches local `make lint` and `npm run build`)
- `npx vitest run` for test execution (matches `npm run test`)

Create the directory first: `mkdir -p .github/workflows`
</action>

</acceptance_criteria>
</task>

<task id="2">
<title>Configure branch protection rule on GitHub</title>
<read_first>
- .github/workflows/ci.yml (confirm the job name used as required status check)
</read_first>
<action>
Configure branch protection for the `main` branch on `Fxl-Business/fxl-core` using the GitHub CLI:

```bash
gh api repos/Fxl-Business/fxl-core/branches/main/protection \
  --method PUT \
  --field "required_status_checks[strict]=true" \
  --field "required_status_checks[contexts][]=Type Check & Test" \
  --field "enforce_admins=true" \
  --field "required_pull_request_reviews=null" \
  --field "restrictions=null"
```

Key settings:
- `required_status_checks.contexts` = `["Type Check & Test"]` ŌĆö matches the `name:` field of the job in ci.yml
- `required_status_checks.strict` = `true` ŌĆö branch must be up to date before merging
- `enforce_admins` = `true` ŌĆö even repo admins cannot bypass the check (matches success criterion: "nao ha como fazer bypass sem desativar a protecao explicitamente")

Note: The status check name must exactly match the `name:` field of the job (not the workflow name). The job is named `Type Check & Test` (from `name: Type Check & Test` under the `type-check-and-test` job key).

If the `gh api` command fails due to permissions, configure manually via:
1. Go to https://github.com/Fxl-Business/fxl-core/settings/branches
2. Click "Add branch protection rule"
3. Branch name pattern: `main`
4. Check "Require status checks to pass before merging"
5. Search and add status check: "Type Check & Test"
6. Check "Require branches to be up to date before merging"
7. Check "Do not allow bypassing the above settings"
8. Save changes
</action>
<acceptance_criteria>
- Running `gh api repos/Fxl-Business/fxl-core/branches/main/protection` returns JSON with `required_status_checks` containing `Type Check & Test`
- The `enforce_admins.enabled` field in the response is `true`
- Attempting to merge a PR with failing CI shows "Required status check has not passed" error on GitHub
</acceptance_criteria>
</task>
