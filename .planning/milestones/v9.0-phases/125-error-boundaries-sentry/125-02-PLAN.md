Modify AppRouter.tsx to wrap module content routes with ModuleErrorBoundary. The approach:

1. Add import at top:
```typescript

```

2. Replace the current `moduleRoutes.map` block inside the Layout route group. Instead of directly rendering module routes, wrap each module's routes in a ModuleErrorBoundary.

Currently the code does:
```typescript
const moduleRoutes = MODULE_REGISTRY
  .flatMap(m => m.routeConfig ?? [])
  .filter((cfg): cfg is RouteObject & { path: string } => cfg.path !== undefined)
```
Then maps them as flat routes.

Change to: Group routes by module and wrap each group. Replace the moduleRoutes approach with per-module wrapping:

```typescript
{MODULE_REGISTRY.map(m => {
  const routes = (m.routeConfig ?? []).filter(
    (cfg): cfg is RouteObject & { path: string } => cfg.path !== undefined,
  )
  if (routes.length === 0) return null
  return routes.map(cfg => (
    <Route
      key={cfg.path}
      path={cfg.path}
      element={
        <ModuleErrorBoundary moduleName={m.label}>
          {cfg.element}
        </ModuleErrorBoundary>
      }
    />
  ))
})}
```

<task id="2">
<title>Add Sentry user and org context enrichment in App.tsx</title>
<read_first>

<action>
Create a lightweight `SentryContextSetter` component that sets Sentry user and org context whenever auth state changes. This component uses hooks (useAuth, useActiveOrg) to read the current user and org, then calls Sentry.setUser and Sentry.setTag.

Add inside App.tsx, BEFORE AppRouter:

```typescript
import * as Sentry from '@sentry/react'
import { useAuth } from '@clerk/react'
import { useEffect } from 'react'
import { useActiveOrg } from '@platform/tenants/useActiveOrg'

function SentryContextSetter() {
  const { userId } = useAuth()
  const { activeOrg } = useActiveOrg()

  useEffect(() => {
    if (userId) {
      Sentry.setUser({ id: userId })
    } else {
      Sentry.setUser(null)
    }
  }, [userId])

  useEffect(() => {
    if (activeOrg?.id) {
      Sentry.setTag('org_id', activeOrg.id)
    } else {
      Sentry.setTag('org_id', undefined)
    }
  }, [activeOrg?.id])

  return null
}
```

Then add `<SentryContextSetter />` inside the App component, right before `<AppRouter />`:

```typescript
export default function App() {
  return (
    <BrowserRouter>
      <ImpersonationProvider>
        <ModuleEnabledProvider>
          <ExtensionProvider>
            <SentryContextSetter />
            <AppRouter />
            <Toaster />
          </ExtensionProvider>
        </ModuleEnabledProvider>
      </ImpersonationProvider>
    </BrowserRouter>
  )
}
```

This ensures:
- All Sentry events include the current user ID
- All Sentry events include the current org_id as a tag
- When org switches, the tag updates automatically
- When user signs out, user context is cleared
- The component renders null (no UI impact)

`SentryContextSetter` must be inside ImpersonationProvider because useActiveOrg depends on it being within the Clerk provider tree (which is in main.tsx).
</action>
