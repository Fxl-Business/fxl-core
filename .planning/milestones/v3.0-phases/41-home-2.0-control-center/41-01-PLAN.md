**Part B — Create module stats hook:**

Create `src/lib/module-stats.ts` with a `useModuleStats` hook that fetches aggregate counts from Supabase for modules that have data tables:

```typescript
export interface ModuleStats {
  /** Map of module ID to badge count */
  counts: Record<string, number>
  loading: boolean
}

export function useModuleStats(): ModuleStats
```

The hook should:
1. Query `tasks` table: `SELECT count(*) FROM tasks WHERE status != 'done'` — this gives the "pending tasks" badge count for the `tasks` module
2. Query `knowledge_entries` table: `SELECT count(*) FROM knowledge_entries` — this gives the total KB entries count for the `knowledge-base` module
3. Return `counts` as `{ 'tasks': N, 'knowledge-base': M }` where N and M are the query results
4. Other modules (docs, ferramentas, clients) return no badge count — their badge stays `undefined`
5. Use `useState` + `useEffect` pattern matching `useActivityFeed`
6. Handle errors silently (tables may not exist in dev environments)
7. Do NOT import from any module directory — only from `@/lib/supabase`

IMPORTANT: The Supabase `.select('*', { count: 'exact', head: true })` pattern returns the count in the response `count` field without fetching rows. Use this efficient pattern.

```typescript
const { count: taskCount } = await supabase
  .from('tasks')
  .select('*', { count: 'exact', head: true })
  .neq('status', 'done')

const { count: kbCount } = await supabase
  .from('knowledge_entries')
  .select('*', { count: 'exact', head: true })
```
