**IMPORTANT: Wave 2 parallel execution handling.**
Plans 03, 04, 05, 06 all run in Wave 2. If they run in parallel, each plan can only reliably update imports for files it OWNS (files_modified). For cross-module imports where the source file is moved by another Wave 2 plan:

Strategy: Each plan updates imports within its OWN files to point to the FINAL @modules/ path. If the target file hasn't been moved yet by another parallel plan, the @modules/ import will still resolve because Plan 03's files are ALREADY in src/modules/docs/ even before Plan 03 runs — they just get more files added.

Wait — the docs components are currently in src/components/docs/, NOT in src/modules/docs/. So if Plan 05 runs before Plan 03, the import `from '@modules/docs/components/MarkdownRenderer'` would FAIL because MarkdownRenderer is still at src/components/docs/.

**SAFE APPROACH:** For cross-module references (DocViewer importing MarkdownRenderer):
- If the target file is still at the OLD location, use the old path.
- The plan that actually MOVES the target file (Plan 03) will update all its consumers.
- So DocViewer.tsx import of MarkdownRenderer should be left as `@/components/docs/MarkdownRenderer` in this plan. Plan 03 will update it when it moves MarkdownRenderer.

In summary: Only update imports where the target is a file that:
1. This plan moves (use new module-relative path), OR
2. Was moved by a Wave 1 plan (use @platform/ or @shared/ path), OR
3. Is staying in place (leave @/ path as-is)

Do NOT update imports pointing to files that other Wave 2 plans move.
