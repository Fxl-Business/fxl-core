Create `tools/sync/sync-up.ts` following the exact same CLI pattern as `tools/wireframe-builder/scripts/generate-blueprint.ts`.

**Script structure:**

1. **Header comment:**
```typescript
/**
 * Sync-up: Import all local .md files from docs/ into Supabase documents table.
 *
 * Reads all .md files recursively, extracts frontmatter, and upserts
 * into the `documents` table using slug as conflict key.
 *
 * NOT a Vite module -- uses process.env, not import.meta.env.
 *
 * Usage:
 *   npx tsx --env-file .env.local tools/sync/sync-up.ts
 */
```

2. **Imports:**
```typescript
import { createClient } from '@supabase/supabase-js'
import { parse as yamlParse } from 'yaml'
import * as fs from 'node:fs'
import * as path from 'node:path'
```

3. **Supabase client (standalone):** Same pattern — process.env, exit if missing.

4. **Helper: `collectMdFiles(dir: string): string[]`**
   - Recursively walks `dir` using `fs.readdirSync` with `{ withFileTypes: true }`
   - For each dirent: if directory, recurse; if file ending in `.md`, push to results
   - Returns array of absolute file paths
   - Do NOT follow symlinks

5. **Helper: `extractFrontmatter(raw: string): { frontmatter: Record<string, string>; body: string }`**
   - Uses same regex as docs-parser: `/^---\n([\s\S]*?)\n---\n([\s\S]*)$/`
   - If no match, return `{ frontmatter: {}, body: raw }`
   - Parse YAML section using `yamlParse()` from the `yaml` package
   - Return `{ frontmatter: parsed, body: bodyPart }`
   - Body part: everything after `---\n`, which may start with `\n` (blank line) — preserve as-is, do NOT trim leading newline (to round-trip correctly with sync-down)

6. **Main function `syncUp()`:**
   - Define `DOCS_DIR` as `path.resolve(process.cwd(), 'docs')`
   - Collect all .md files: `collectMdFiles(DOCS_DIR)`
   - If empty, log "No .md files found in docs/" and exit
   - Build array of document records to upsert:
     For each file path:
     a. Read file content: `fs.readFileSync(filePath, 'utf-8')`
     b. Extract frontmatter and body
     c. Derive slug: remove DOCS_DIR prefix, remove `.md` suffix, normalize separators
        Example: `/path/to/project/docs/processo/fases/fase1.md` -> `processo/fases/fase1`
     d. Derive parent_path: directory portion of slug + trailing `/`
        Example: `processo/fases/fase1` -> `processo/fases/`
        For root-level files (e.g., slug `index`): parent_path is empty string `''` (but there are currently no root-level docs, so this is edge-case handling)
     e. Build record:
        ```typescript
        {
          title: frontmatter.title || path.basename(filePath, '.md'),
          badge: frontmatter.badge || '',
          description: frontmatter.description || '',
          slug,
          parent_path: parentPath,
          body,
          updated_at: new Date().toISOString(),
        }
        ```
     f. Do NOT include `sort_order` in the upsert — let existing sort_order be preserved for existing rows, and new rows will get the default (0). This avoids overwriting manually set ordering.
     g. Do NOT include `id` or `created_at` — let the database handle these via defaults.

   - Upsert all records. Use batched approach for reliability — process one at a time to get clear error messages per document:
     ```typescript
     let successCount = 0
     let errorCount = 0
     for (const record of records) {
       const { error } = await supabase
         .from('documents')
         .upsert(record, { onConflict: 'slug' })
       if (error) {
         console.error(`Error upserting "${record.slug}": ${error.message}`)
         errorCount++
       } else {
         successCount++
       }
     }
     ```
   - Log summary: `console.log(\`Synced ${successCount} documents to Supabase (${errorCount} errors)\`)`
   - If any errors, `process.exit(1)`

**Key details:**
- Frontmatter parsing uses `yamlParse` from the `yaml` package (same package used by the app's docs-parser)
- Body content is sent to Supabase exactly as read from file (custom tags preserved verbatim)
- Slug derivation uses `path.relative(DOCS_DIR, filePath)` and replaces `.md` suffix and normalizes path separators (important for Windows compatibility, though primary target is macOS)
- The `onConflict: 'slug'` ensures idempotent upserts — running sync-up twice produces no duplicates
- NOT including sort_order in upsert preserves any manual ordering set in the DB
- If a file has no frontmatter (no `---` block), use filename as title and empty strings for badge/description

IMPORTANT: This script does NOT import from `@/` or `src/` — standalone Node.js script. All imports from npm packages or `node:*` built-ins.
