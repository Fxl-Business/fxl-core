Create `tools/sync/sync-down.ts` following the exact same CLI pattern as `tools/wireframe-builder/scripts/generate-blueprint.ts`.

**Script structure:**

1. **Header comment:**
```typescript
/**
 * Sync-down: Export all documents from Supabase to local .md files.
 *
 * Reads all rows from the `documents` table, reconstructs .md files
 * with YAML frontmatter in the docs/ directory.
 *
 * NOT a Vite module -- uses process.env, not import.meta.env.
 *
 * Usage:
 *   npx tsx --env-file .env.local tools/sync/sync-down.ts
 */
```

2. **Imports:**
```typescript
import { createClient } from '@supabase/supabase-js'
import { stringify as yamlStringify } from 'yaml'
import * as fs from 'node:fs'
import * as path from 'node:path'
```

3. **Supabase client (standalone):** Same pattern as generate-blueprint.ts — read from `process.env.VITE_SUPABASE_URL` and `process.env.VITE_SUPABASE_PUBLISHABLE_KEY`, exit with error message if missing.

4. **Type for document row:**
```typescript
type DocumentRow = {
  title: string
  badge: string
  description: string
  slug: string
  parent_path: string
  body: string
  sort_order: number
}
```

5. **Main function `syncDown()`:**
   - Fetch all documents: `supabase.from('documents').select('title, badge, description, slug, parent_path, body, sort_order').order('parent_path').order('sort_order')`
   - If error, `console.error()` + `process.exit(1)`
   - If no data or empty array, log "No documents found in Supabase." and exit
   - Define `DOCS_DIR` as `path.resolve(process.cwd(), 'docs')`
   - For each document row:
     a. Build file path: `path.join(DOCS_DIR, doc.slug + '.md')`
     b. Create parent directory: `fs.mkdirSync(path.dirname(filePath), { recursive: true })`
     c. Build frontmatter object: `{ title: doc.title, badge: doc.badge }` — add `description` only if non-empty string
     d. Build file content: `---\n${yamlStringify(frontmatter).trim()}\n---\n\n${doc.body}`
     e. Write file: `fs.writeFileSync(filePath, content, 'utf-8')`
   - Log summary: `console.log(\`Synced ${data.length} documents from Supabase to docs/\`)`

**Key details:**
- Use `yamlStringify` from the `yaml` package (already a project dependency) to generate frontmatter — this ensures correct YAML escaping for titles with special characters (em dashes, colons, etc.)
- Only include `description` in frontmatter if it's a non-empty string (some docs don't have it, and the column defaults to '')
- Body is written exactly as stored in the DB — custom tags are preserved verbatim
- The trailing newline in body should be preserved (write body as-is, do NOT trim)
- Do NOT delete files that exist locally but not in the DB — sync-down only creates/updates

6. **Execute:** `syncDown().catch(err => { console.error('Unexpected error:', err); process.exit(1) })`

IMPORTANT: This script does NOT import from `@/` or `src/` — it's a standalone Node.js script. All imports are from npm packages or `node:*` built-ins.
