**Activity Feed (HOME-02):**
- Create a `useActivityFeed()` hook INSIDE Home.tsx (single use, no separate file needed).
- The hook fetches from `knowledge_entries` and `tasks` tables via Supabase using Promise.all (two parallel `.from()` calls, each `.order('updated_at', { ascending: false }).limit(10)`).
- Merge both result arrays into a unified `ActivityItem[]` type: `{ id, title, type: 'kb_entry' | 'task', subtype, client_slug, updated_at, href }`.
- Use the `mergeAndSortActivityItems` function pattern from Plan 33-00's test (merge, sort by updated_at descending, slice first 10). Export the merge function so the test in Home.test.tsx can import it. Update the test import path from the test file's local definition to `@/pages/Home`.
- Sort merged array by `updated_at` descending, slice first 10.
- Wrap Promise.all in try/catch -- on error, set items to empty array (tables may not exist in dev).
- Use `cancelled` flag pattern in useEffect cleanup to avoid state updates on unmounted component.
- CRITICAL: Call `useActivityFeed()` unconditionally at the TOP of the Home component, BEFORE any conditional returns (hooks-before-early-returns rule from project memory).

**Activity Feed Rendering:**
- Section header: "Atividade recente" in the same 10px uppercase bold style.
- Show loading state: "Carregando..." text while loading.
- Show empty state: "Nenhuma atividade recente." when items is empty and not loading.
- Each activity item: a row showing an icon (BookOpen for kb_entry, CheckSquare for task), title as link, subtype badge (entry_type or status), client_slug if present, and formatted date (toLocaleDateString('pt-BR')).
- For kb_entry items: `href: /knowledge-base/${id}`. For task items: `href: /tarefas`.

**Keep the existing Clientes section** (the `clients` array with Financeiro Conta Azul) -- this is still needed and not replaced by the module grid. Render it below the activity feed.
