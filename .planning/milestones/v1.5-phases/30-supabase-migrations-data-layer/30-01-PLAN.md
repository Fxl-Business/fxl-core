  <files>supabase/migrations/005_knowledge_entries.sql</files>
  <action>
Create SQL migration file `supabase/migrations/005_knowledge_entries.sql` with:

1. CREATE TABLE public.knowledge_entries:
   - id: uuid PRIMARY KEY DEFAULT gen_random_uuid()
   - entry_type: text NOT NULL CHECK (entry_type IN ('bug', 'decision', 'pattern', 'lesson'))
   - title: text NOT NULL
   - body: text NOT NULL
   - tags: text[] NOT NULL DEFAULT '{}'
   - client_slug: text (nullable)
   - created_by: text (nullable, Clerk user ID)
   - created_at: timestamptz NOT NULL DEFAULT now()
   - updated_at: timestamptz NOT NULL DEFAULT now()
   - search_vec: tsvector GENERATED ALWAYS AS (to_tsvector('portuguese', coalesce(title, '') || ' ' || coalesce(body, ''))) STORED

2. ENABLE ROW LEVEL SECURITY

3. Four anon-permissive RLS policies (SELECT, INSERT, UPDATE, DELETE) — all using `TO anon USING (true)` / `WITH CHECK (true)`. Include DELETE policy (unlike 004 which omitted it) because KB entries should be deletable.

4. Three indexes:
   - GIN index on search_vec: `idx_knowledge_entries_search`
   - B-tree on client_slug: `idx_knowledge_entries_client_slug`
   - B-tree on entry_type: `idx_knowledge_entries_entry_type`

Locked decisions from STATE.md:
- Column name is `entry_type` (not kind or category)
- tsvector language is 'portuguese'
- Table name is `knowledge_entries` (per ROADMAP success criteria)

<task type="auto">
  <name>Task 2: Create migration 006 — tasks table and apply both migrations</name>

  <action>
Create SQL migration file `supabase/migrations/006_tasks.sql` with:

1. CREATE TABLE public.tasks:
   - id: uuid PRIMARY KEY DEFAULT gen_random_uuid()
   - title: text NOT NULL
   - description: text NOT NULL DEFAULT ''
   - status: text NOT NULL DEFAULT 'todo' CHECK (status IN ('todo', 'in_progress', 'done', 'blocked'))
   - priority: text NOT NULL DEFAULT 'medium' CHECK (priority IN ('low', 'medium', 'high'))
   - client_slug: text (nullable)
   - due_date: date (nullable)
   - created_by: text (nullable, Clerk user ID)
   - created_at: timestamptz NOT NULL DEFAULT now()
   - updated_at: timestamptz NOT NULL DEFAULT now()

2. ENABLE ROW LEVEL SECURITY

3. Four anon-permissive RLS policies (SELECT, INSERT, UPDATE, DELETE)

4. Two indexes:
   - B-tree on client_slug: `idx_tasks_client_slug`
   - B-tree on status: `idx_tasks_status`

After creating both migration files, apply them via `make migrate`. Verify in the command output that both migrations were applied successfully.
