  <action>
Create a single form component that handles both create (`/tarefas/new`) and edit (`/tarefas/:id/edit`) modes.

**Mode detection:** Use `useParams<{ id?: string }>()`. If `id` is present, it's edit mode — fetch existing task via `getTask(id)` in useEffect and populate form state.

**ALL hooks MUST be at the top of the component before any conditional returns** (critical rule from MEMORY.md — caused 2 crashes before).

**Form fields (all controlled via useState):**
1. `title` (string) — shadcn Input, required, placeholder "Titulo da tarefa"
2. `description` (string) — shadcn Textarea, optional, placeholder "Descricao", rows={4}
3. `status` (TaskStatus) — shadcn Select with STATUS_LABELS options, default 'todo'
4. `priority` (TaskPriority) — shadcn Select with PRIORITY_LABELS options, default 'medium'
5. `due_date` (string) — plain `<input type="date" />` styled with shadcn Input classes. Keep as string "YYYY-MM-DD" (PostgreSQL date column returns string, not Date object). Default empty string.
6. `client_slug` (string) — shadcn Input, optional, placeholder "Slug do cliente (ex: financeiro-conta-azul)"

**Layout:** Form wrapped in a max-w-2xl mx-auto container. Page header with "Nova Tarefa" or "Editar Tarefa" title. Two-column grid for status+priority row. Full-width for title, description, due_date, client_slug. Submit button at bottom — "Criar Tarefa" or "Salvar Alteracoes" depending on mode, disabled while saving.

**Submit handler:**
- Set `saving=true`
- Call `createTask(...)` or `updateTask(id, ...)` from tasks-service
- On success: `toast.success('Tarefa criada'|'Tarefa atualizada')` via sonner, then `navigate('/tarefas')`
- On error: `toast.error('Erro ao salvar tarefa')`, set saving=false
- Pass `due_date: dueDate || null` and `client_slug: clientSlug || null` (empty string becomes null for Supabase)

**Back link:** A "Voltar" text button/link at the top that navigates to `/tarefas`.

**Loading state for edit mode:** Show Loader2 spinner while fetching existing task data.

If Plan 01 registered the route with a lazy import placeholder, replace that placeholder with the actual TaskForm import in App.tsx. If App.tsx already imports TaskForm correctly, no change needed.
