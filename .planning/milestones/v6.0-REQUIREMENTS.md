# Requirements: Nexo v6.0 — Reestruturação de Módulos

**Defined:** 2026-03-18
**Shipped:** 2026-03-18
**Core Value:** Separar responsabilidades (cadastro vs criação) e dar a cada módulo seu próprio workspace na sidebar

## v6.0 Requirements

### Clientes

- [x] **CLI-01**: Operador pode ver lista de clientes da org (cards ou tabela)
- [x] **CLI-02**: Operador pode criar cliente com nome, slug e status
- [x] **CLI-03**: Operador pode ver perfil de um cliente (nome, logo, status, link para projetos)
- [x] **CLI-04**: Operador pode editar dados de um cliente existente
- [x] **CLI-05**: Sidebar mostra lista dinâmica de clientes da org quando workspace Clientes está ativo

### Projetos

- [x] **PROJ-01**: Operador pode ver lista de projetos da org
- [x] **PROJ-02**: Operador pode criar projeto vinculado a um cliente ou à própria org
- [x] **PROJ-03**: Operador pode acessar briefing em /projetos/:slug/briefing
- [x] **PROJ-04**: Operador pode acessar blueprint em /projetos/:slug/blueprint
- [x] **PROJ-05**: Operador pode acessar wireframe em /projetos/:slug/wireframe
- [x] **PROJ-06**: Operador pode acessar branding em /projetos/:slug/branding
- [x] **PROJ-07**: Sidebar mostra lista dinâmica de projetos (expandível com sub-itens) quando workspace Projetos está ativo

### Sidebar Workspace

- [x] **SIDE-01**: Sidebar tem dropdown switcher no topo para trocar módulo ativo
- [x] **SIDE-02**: Trocar módulo no dropdown muda rota para home do módulo e carrega nav daquele módulo
- [x] **SIDE-03**: ModuleDefinition suporta useNavItems() hook opcional para nav dinâmico
- [x] **SIDE-04**: Footer fixo da sidebar mostra Admin (se super_admin) e Perfil

### Migração de Dados

- [x] **DATA-01**: Tabela projects criada com id, slug, name, client_id nullable, org_id + RLS
- [x] **DATA-02**: Tabela clients estendida com logo_url e status
- [x] **DATA-03**: Tabelas briefing_configs, blueprint_configs, comments, share_tokens ganham coluna project_id
- [x] **DATA-04**: Dados existentes migrados: projeto "Financeiro Conta Azul" criado e FKs atualizadas
- [x] **DATA-05**: Stores (briefing-store, blueprint-store, comments, tokens) migrados de client_slug para project_id

### Código

- [x] **CODE-01**: src/modules/clients/ renomeado para src/modules/projects/ com module ID 'projects'
- [x] **CODE-02**: Novo src/modules/clients/ criado do zero (CRUD simples)
- [x] **CODE-03**: Pages hardcoded de FinanceiroContaAzul substituídas por pages genéricas parametrizadas por slug
- [x] **CODE-04**: module-ids.ts atualizado com PROJECTS e CLIENTS com novo significado

## Out of Scope

| Feature | Reason |
|---------|--------|
| CRM completo | Será app separada — Nexo é gestão/entrega, não operação comercial |
| Redirect de URLs antigas (/clients/:slug/*) | Rotas internas, não públicas. /wireframe-view?token=... permanece intocado |
| Remoção de client_slug das tabelas | Mantido temporariamente para backward compat, removido em migration futura |
| RLS restritivo em briefing_configs/blueprint_configs | Considerar em milestone futuro (atualmente USING(true)) |
| Drag-and-drop de projetos/clientes | Não necessário para v6.0 |
